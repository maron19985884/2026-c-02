# Feature Specification: オンライン書店 購買フロー

**Feature Branch**: `001-purchase-flow`

**Created**: 2026-06-30

**Status**: Draft

**Input**: User description: "オンライン書店の購買フロー（商品一覧・商品詳細・カート・注文フォーム・注文完了の5画面）を実装する。ログイン・決済・在庫管理・管理画面はスコープ外。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 書籍一覧の閲覧 (Priority: P1)

購入希望者が書店サイトを訪れ、販売中の書籍をグリッド形式で一覧表示し、気になる書籍の詳細画面へ遷移できる。

**Why this priority**: 全購買フローの起点であり、これがなければ他のすべての機能が利用できない。最小限のMVPとして書籍一覧の閲覧が必要。

**Independent Test**: 書籍一覧画面を表示し、書籍の書影・タイトル・著者・価格が確認できること。書籍をクリックすると詳細画面に遷移できること。

**Acceptance Scenarios**:

1. **Given** 書店サイトにアクセスした状態で, **When** 商品一覧画面を開くと, **Then** 販売中の書籍がグリッド形式で複数表示され、各書籍の書影・タイトル・著者・価格が確認できる
2. **Given** 商品一覧画面が表示されている状態で, **When** 任意の書籍をクリックすると, **Then** その書籍の詳細画面へ遷移する

---

### User Story 2 - 書籍詳細の確認とカートへの追加 (Priority: P2)

購入希望者が書籍詳細画面で書影・タイトル・著者・価格・説明文を確認し、「カートに追加」ボタンを押して書籍をカートに追加できる。追加後も他の書籍を閲覧し続けられる。

**Why this priority**: 購入意思決定に必要な詳細情報の提供と、カートへの追加というコアアクションを実現する。一覧だけではユーザーが購買判断できない。

**Independent Test**: 特定の書籍詳細画面で書籍情報が表示されること。「カートに追加」ボタンを押してカートに追加され、商品一覧画面に戻れること。

**Acceptance Scenarios**:

1. **Given** 商品一覧から書籍をクリックした状態で, **When** 詳細画面が表示されると, **Then** 書影・タイトル・著者・価格・説明文がすべて表示される
2. **Given** 書籍詳細画面で, **When** 「カートに追加」ボタンを押すと, **Then** 書籍がカートに追加され、引き続き一覧画面へ戻って他の書籍を閲覧できる

---

### User Story 3 - カートの管理 (Priority: P3)

購入希望者がカートに入れた書籍の一覧を確認し、数量の変更・削除ができ、合計金額がリアルタイムに更新され、注文手続きに進める。

**Why this priority**: カートが機能しなければ購入を完了できない。数量変更・削除・合計金額の確認はすべてカート画面に集約される。

**Independent Test**: カートに書籍が入っている状態で、数量変更・削除・合計金額の更新が即時反映され、注文フォームへ遷移できること。

**Acceptance Scenarios**:

1. **Given** カートに書籍が追加されている状態で, **When** カート画面を開くと, **Then** 書名・単価・数量・小計・全商品の合計金額が表示される
2. **Given** カート画面で, **When** 書籍の数量を変更すると, **Then** その書籍の小計と全商品の合計金額がリアルタイムに更新される
3. **Given** カート画面で, **When** 書籍の削除ボタンを押すと, **Then** その書籍がカートから削除され、合計金額が即時更新される
4. **Given** カートに1冊以上の書籍がある状態で, **When** 「注文手続きへ」ボタンを押すと, **Then** 注文フォーム画面へ遷移する

---

### User Story 4 - 注文の確定と完了 (Priority: P4)

購入希望者が氏名・住所・メールアドレスを入力し、カート内容を確認しながら注文を確定し、注文完了画面で注文番号を受け取り、商品一覧に戻れる。

**Why this priority**: 購買フロー全体のゴールである注文完了を実現する。入力バリデーション・注文確定・完了通知をすべて含む。

**Independent Test**: 注文フォーム画面で必須項目（氏名・住所・メールアドレス）を入力し「注文する」ボタンを押すと、注文完了画面に注文番号が表示されること。未入力の場合はエラーが表示されること。

**Acceptance Scenarios**:

1. **Given** 注文フォーム画面で, **When** 氏名・住所・メールアドレスをすべて正しく入力して「注文する」ボタンを押すと, **Then** 注文完了画面に遷移し、注文完了メッセージと注文番号が表示される
2. **Given** 注文フォーム画面で, **When** 必須項目のいずれかが未入力の状態で「注文する」を押すと, **Then** 対象項目にエラーメッセージが表示され、画面遷移しない
3. **Given** メールアドレス欄に不正な形式の文字列を入力した状態で, **When** 「注文する」を押すと, **Then** 形式エラーのメッセージが表示される
4. **Given** 注文フォーム画面で, **When** 画面を開くと, **Then** カートに追加した注文商品と合計金額が同画面で確認できる
5. **Given** 注文完了画面で, **When** 「商品一覧に戻る」リンクをクリックすると, **Then** 商品一覧画面へ遷移する

---

### Edge Cases

- カートが空の状態で注文フォームに直接アクセスした場合、カート画面にリダイレクトされる
- カート内の数量を0にしようとした場合、削除と同等の動作をする（または1未満への変更を禁止する）
- 注文フォームのメールアドレスが正しい形式でない場合（例：@がない）、形式エラーを表示する
- 注文完了後にブラウザの「戻る」ボタンを押しても同じ注文が二重送信されない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは販売中の書籍をグリッド形式で一覧表示しなければならない
- **FR-002**: 書籍一覧には各書籍の書影・タイトル・著者・価格を表示しなければならない
- **FR-003**: ユーザーは書籍をクリックすることで詳細画面に遷移できなければならない
- **FR-004**: 書籍詳細画面には書影・タイトル・著者・価格・説明文を表示しなければならない
- **FR-005**: ユーザーは「カートに追加」ボタン1つで書籍をカートに追加できなければならない
- **FR-006**: カートに追加後も商品一覧画面に戻り他の書籍を閲覧し続けられなければならない
- **FR-007**: カート画面には追加した書籍の書名・単価・数量・小計を一覧表示しなければならない
- **FR-008**: カート画面には全書籍の合計金額を表示しなければならない
- **FR-009**: 数量変更により小計と合計金額がリアルタイムに更新されなければならない
- **FR-010**: ユーザーはカートから書籍を削除でき、削除後に合計金額が即時更新されなければならない
- **FR-011**: ユーザーはカート画面から注文フォーム画面へ遷移できなければならない
- **FR-012**: 注文フォームには氏名・住所・メールアドレスの3項目があり、すべて必須入力でなければならない
- **FR-013**: 注文フォームには注文商品と合計金額が同画面で表示されなければならない
- **FR-014**: 未入力や形式不正の場合はエラーメッセージを表示しなければならない
- **FR-015**: 「注文する」ボタン1つで注文を確定し注文完了画面に遷移できなければならない
- **FR-016**: 注文完了画面には注文完了メッセージと注文番号を表示しなければならない
- **FR-017**: 注文完了画面から商品一覧へ戻るリンクがなければならない

### Key Entities

- **書籍 (Book)**: 書影・タイトル・著者・価格・説明文を持つ販売商品
- **カートアイテム (CartItem)**: ユーザーがカートに追加した書籍と数量の組み合わせ
- **注文 (Order)**: 注文者情報（氏名・住所・メールアドレス）と注文書籍一覧・合計金額
- **注文番号 (OrderNumber)**: 各注文を一意に識別するシステム採番の番号

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーが書店サイトにアクセスしてから注文完了まで、5分以内に操作を完了できる
- **SC-002**: 5画面（商品一覧・詳細・カート・注文フォーム・注文完了）すべてが操作後3秒以内に表示される
- **SC-003**: カートの数量変更後1秒以内に小計・合計金額が更新される
- **SC-004**: 必須項目が未入力または形式不正の状態で注文を試みた場合、100%エラーメッセージが表示される
- **SC-005**: 注文確定ごとに一意の注文番号が100%の確率で発行・表示される
- **SC-006**: 5画面すべての画面遷移（一覧→詳細→カート→注文フォーム→完了→一覧）が正常に機能する

## Assumptions

- ユーザーはブラウザからアクセスし、ログインや会員登録は不要（認証なし）
- 書籍データは事前に登録済みのものを表示する（管理画面・在庫管理はスコープ外）
- 送料・税金の計算はスコープ外（合計金額は書籍価格 × 数量の合算のみ）
- 決済処理はスコープ外（「注文する」ボタン押下をもって注文確定とみなす）
- カートのデータはブラウザセッション中のみ保持されれば十分（永続化・ログイン連携は不要）
- 注文番号はシステムが自動採番する
- メールアドレスの形式チェックは実施するが、実際のメール送信はスコープ外
- 検索・フィルター・レビュー・評価機能はスコープ外
