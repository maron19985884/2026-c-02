# Quickstart: 購買フロー全体（5画面）の動作確認

**Feature**: 001-purchase-flow
**Date**: 2026-06-28

## 前提条件

- Docker Compose が起動していること (`docker-compose up -d`)
- frontend: http://localhost:3000
- backend: http://localhost:4000
- mysql: localhost:3306
- books テーブルに3件以上のサンプルデータが登録されていること

## エンドツーエンドシナリオ（正常系フル購買フロー）

### Step 1: 商品一覧の確認

1. `http://localhost:3000/` を開く
2. 書籍カードがグリッド形式で複数表示されること
3. 各カードに書影・タイトル・著者・価格が表示されていること

**期待結果**: ✅ 書籍一覧が表示される

### Step 2: 商品詳細の確認

1. 一覧から任意の書籍カードをクリック
2. `http://localhost:3000/books/{id}` へ遷移すること
3. 書影・タイトル・著者・価格・説明文が表示されていること
4. 「カートに追加」ボタンが表示されていること

**期待結果**: ✅ 書籍詳細が表示される

### Step 3: カートへの追加

1. 「カートに追加」をクリック
2. 商品一覧（`/`）へ遷移すること
3. 別の書籍を選んで詳細画面へ遷移し、再度「カートに追加」をクリック
4. 同じ書籍を再度カートに追加し、数量が2になることを確認する（重複チェック）

**期待結果**: ✅ カートに2種類の書籍が入り、同一書籍は数量加算される

### Step 4: カートの確認・調整

1. カートアイコンまたはナビゲーションからカート画面（`/cart`）へ遷移
2. 書名・単価・数量・小計・合計金額が表示されること
3. 数量増減ボタンで数量を変更し、小計・合計がリアルタイム更新されること
4. 1件の書籍を削除し、合計が即時更新されること
5. 「注文手続きへ」ボタンが表示されていること

**期待結果**: ✅ カート操作が正常に動作する

### Step 5: 注文フォーム入力

1. 「注文手続きへ」をクリックして注文フォーム（`/order`）へ遷移
2. カート内の書名・単価・数量・小計・合計金額が同画面に表示されること
3. 氏名: `山田 太郎`、住所: `東京都渋谷区xxx-xxx`、メール: `yamada@example.com` を入力
4. 「注文する」をクリック

**期待結果**: ✅ 注文完了画面へ遷移する

### Step 6: 注文完了の確認

1. `/order/complete` へ遷移していること
2. 注文完了メッセージと注文番号（例: `ORD-20260628-000001`）が表示されること
3. 「商品一覧へ戻る」リンクをクリックすると `/` へ遷移すること

**期待結果**: ✅ 注文番号が表示され、一覧へ戻れる

---

## 個別シナリオ

### バリデーション（未入力）

1. 注文フォームを開き全項目空で「注文する」をクリック
2. 氏名・住所・メールの3フィールドにエラーメッセージが表示されること

### バリデーション（メール形式不正）

1. メール欄に `test-email`（`@` なし）を入力して「注文する」
2. 「メールアドレスの形式が正しくありません」が表示されること

### 空カートリダイレクト

1. カートを空にした状態で `http://localhost:3000/order` へ直接アクセス
2. カート画面（`/cart`）へリダイレクトされること

## API 直接テスト (curl)

```bash
# 書籍一覧取得
curl http://localhost:4000/api/books

# 書籍詳細取得
curl http://localhost:4000/api/books/1

# 注文確定（正常系）
curl -X POST http://localhost:4000/api/orders \
  -H "Content-Type: application/json" \
  -d '{
    "customer_name": "山田 太郎",
    "customer_address": "東京都渋谷区xxx-xxx",
    "customer_email": "yamada@example.com",
    "items": [{"book_id": 1, "quantity": 1, "unit_price": 2800}]
  }'
# 期待: 201 Created, {"order_number": "ORD-..."}
```
