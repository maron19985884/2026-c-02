# Feature Specification: 購買フロー全体

**Feature Branch**: `feature/001-purchase-flow`

**Created**: 2026-06-28

**Status**: Draft

**Input**: User description: "購買フロー全体（商品一覧〜注文完了の5画面）"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 書籍を探してカートに追加する (Priority: P1)

購入希望者が販売中の書籍を一覧で閲覧し、興味を持った書籍の詳細を確認したうえでカートに追加できる。

**Why this priority**: 購買フローの起点であり、この流れが機能しなければ後続のすべての画面が無意味になる。

**Independent Test**: 商品一覧を開き、任意の書籍をクリックして詳細を確認し、「カートに追加」を押すことで単独テスト可能。カートに書籍が追加されれば成功。

**Acceptance Scenarios**:

1. **Given** 書籍が販売されている状態で、**When** 商品一覧ページを開くと、**Then** 書影・タイトル・著者・価格がグリッド形式で表示される
2. **Given** 商品一覧が表示されている状態で、**When** 書籍カードをクリックすると、**Then** 書影・タイトル・著者・価格・説明文が表示された詳細画面へ遷移する
3. **Given** 書籍詳細画面を表示している状態で、**When** 「カートに追加」ボタンを押すと、**Then** 書籍がカートに追加され、商品一覧画面へ戻る
4. **Given** 同一書籍がすでにカートにある状態で、**When** 再度「カートに追加」を押すと、**Then** 新規追加ではなく数量が1加算される

---

### User Story 2 - カートの内容を確認・調整する (Priority: P2)

購入希望者がカート画面で注文予定の書籍一覧を確認し、数量の増減や不要な書籍の削除を行い、合計金額を把握できる。

**Why this priority**: カートへの追加（P1）が完成した後に単独で機能するMVP拡張であり、購入意思決定の場となる重要な画面。

**Independent Test**: カートに書籍を追加した状態でカート画面を開き、数量変更・削除・合計金額確認を実施することで単独テスト可能。

**Acceptance Scenarios**:

1. **Given** カートに書籍が入っている状態で、**When** カート画面を開くと、**Then** 書名・単価・数量・小計・合計金額が一覧表示される
2. **Given** カート画面を表示している状態で、**When** 数量を増減すると、**Then** 小計と合計金額がリアルタイムに更新される
3. **Given** カート画面を表示している状態で、**When** 削除ボタンを押すと、**Then** 該当書籍がカートから除去され合計金額が即時更新される
4. **Given** カート画面を表示している状態で、**When** 「注文手続きへ」ボタンを押すと、**Then** 注文フォーム画面へ遷移する

---

### User Story 3 - 注文情報を入力して注文を確定する (Priority: P3)

購入希望者が注文フォームに氏名・住所・メールアドレスを入力して「注文する」ボタンを押し、注文を確定して注文番号を受け取る。

**Why this priority**: P1・P2が完成した後に追加するMVP完成フェーズ。注文フォーム単体でもカートを前提として動作確認できる。

**Independent Test**: カートに書籍を追加した状態で注文フォームを開き、必要事項を入力して「注文する」を押すことで単独テスト可能。注文完了画面と注文番号が表示されれば成功。

**Acceptance Scenarios**:

1. **Given** カートに書籍がある状態で注文フォームを開いたとき、**When** フォームが表示されると、**Then** カート内の書名・単価・数量・小計と合計金額も同画面に表示される
2. **Given** 注文フォームを入力している状態で、**When** 全項目を正しく入力して「注文する」ボタンを押すと、**Then** 注文が確定し注文完了画面へ遷移する
3. **Given** 注文完了画面が表示された状態で、**When** 画面を確認すると、**Then** 注文完了メッセージと一意の注文番号が表示される
4. **Given** 注文完了画面が表示された状態で、**When** 「商品一覧へ戻る」リンクをクリックすると、**Then** 商品一覧画面へ遷移する
5. **Given** 注文フォームを入力している状態で、**When** 必須項目が未入力のまま「注文する」を押すと、**Then** 該当項目の下にエラーメッセージが表示され、注文処理は実行されない
6. **Given** 注文フォームを入力している状態で、**When** メールアドレスの形式が不正なまま「注文する」を押すと、**Then** 「メールアドレスの形式が正しくありません」と表示される
7. **Given** カートが空の状態で、**When** 注文フォームのURLへ直接アクセスすると、**Then** カート画面へリダイレクトされる

---

### Edge Cases

- カートが空の状態で注文フォームURLへ直接アクセスした場合はカート画面へリダイレクトする
- 同一書籍を複数回カートに追加した場合は数量を加算し、重複エントリは作成しない
- メールアドレスが形式上正しくない場合（`@`なし等）はメール形式エラーを表示する
- 氏名・住所・メールのいずれかが未入力のまま送信しようとした場合は注文を受け付けない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ユーザーは販売中の書籍を書影・タイトル・著者・価格とともにグリッド形式の一覧で閲覧できなければならない
- **FR-002**: ユーザーは書籍一覧から書籍を選択して、書影・タイトル・著者・価格・説明文を含む詳細情報を確認できなければならない
- **FR-003**: ユーザーは書籍詳細画面から書籍をカートに追加でき、追加後は商品一覧へ戻ることができなければならない
- **FR-004**: 同一書籍を複数回カートに追加した場合、数量を加算しなければならない（重複エントリ禁止）
- **FR-005**: ユーザーはカート画面で書名・単価・数量・小計・合計金額を確認できなければならない
- **FR-006**: ユーザーはカート内の書籍数量を増減でき、小計と合計金額がリアルタイムに更新されなければならない
- **FR-007**: ユーザーはカートから任意の書籍を削除でき、削除後に合計金額が即時更新されなければならない
- **FR-008**: ユーザーは氏名・住所・メールアドレスを注文フォームに入力して注文を確定できなければならない（全項目必須）
- **FR-009**: 注文フォームで必須項目が未入力またはメールアドレスの形式が不正な場合、該当項目にエラーメッセージを表示し注文処理を中断しなければならない
- **FR-010**: 注文確定時にシステムは一意の注文番号を発行しなければならない
- **FR-011**: 注文完了画面では注文完了メッセージ・注文番号・商品一覧へ戻るリンクを表示しなければならない
- **FR-012**: カートが空の状態で注文フォームへアクセスした場合、カート画面へリダイレクトしなければならない

### Key Entities *(include if feature involves data)*

- **書籍 (Book)**: 販売対象の書籍。書影・タイトル・著者・価格・説明文を持つ
- **カート (Cart)**: セッション単位で管理される購入前の一時的な書籍置き場。複数のカートアイテムを保持する
- **カートアイテム (CartItem)**: カート内の1レコード。書籍・数量・小計の情報を持つ
- **注文 (Order)**: 注文確定時に生成される記録。一意の注文番号・顧客情報（氏名・住所・メール）・注文アイテムリストを持つ
- **注文アイテム (OrderItem)**: 注文に含まれる書籍の記録。書籍・数量・単価を保持する

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーが商品一覧から注文完了画面に到達するまでの操作を5分以内に完了できる
- **SC-002**: 各画面（一覧・詳細・カート・注文フォーム・注文完了）の表示が3秒以内に完了する
- **SC-003**: 注文フォームで必須項目が未入力の場合、100%のケースでエラーメッセージが表示され注文が処理されない
- **SC-004**: 注文確定のたびに一意の注文番号が発行され、重複が発生しない
- **SC-005**: カートの数量変更・削除操作後、合計金額が即時（遅延なし）に更新される
- **SC-006**: 同一書籍をカートに複数回追加した場合、常に数量加算として処理され重複エントリが作成されない

## Assumptions

- カートの保持はセッション単位とし、ブラウザを閉じると消去される（永続化はスコープ外）
- 送料は合計金額に含めない（スコープ外）
- ユーザー認証は実装しない（ゲスト購入のみ）
- 決済処理は実装しない（注文確定のみ行う）
- 書籍データはシステムにあらかじめ登録されており、ユーザーが追加・変更することはない
- 在庫管理はスコープ外のため、在庫数のチェックは行わない
- 注文番号はバックエンドが採番し、フロントエンドに返す
- メールアドレスの形式チェックは「`@`を含む標準的なメール形式」を基準とする
