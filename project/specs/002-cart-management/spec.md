# Feature Specification: カート機能

**Feature Branch**: `feature/002-cart-management`

**Created**: 2026-06-28

**Status**: Draft

**Input**: User description: "カート機能（追加・数量変更・削除・合計金額表示）"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 書籍をカートに追加する (Priority: P1)

購入希望者が書籍詳細画面から「カートに追加」ボタンを押して書籍をカートに追加できる。
すでにカートに同じ書籍がある場合は数量を加算する。

**Why this priority**: カート機能の起点であり、追加ができなければ数量変更・削除・合計表示のすべてが意味をなさない。

**Independent Test**: 書籍詳細画面を開き「カートに追加」を押すことで単独テスト可能。
カートに書籍が1件追加されること、または既存エントリの数量が1加算されることを確認する。

**Acceptance Scenarios**:

1. **Given** カートが空の状態で、**When** 書籍詳細画面の「カートに追加」ボタンを押すと、**Then** 書籍がカートに1件追加され、商品一覧画面へ戻る
2. **Given** ある書籍がカートに1件入っている状態で、**When** 同じ書籍の詳細画面で「カートに追加」を押すと、**Then** 新しいエントリが増えるのではなく、既存エントリの数量が2になる
3. **Given** 異なる書籍Aがすでにカートにある状態で、**When** 書籍Bの詳細画面で「カートに追加」を押すと、**Then** カートに書籍Aと書籍Bの2件が登録される

---

### User Story 2 - カート内容を確認し数量を調整する (Priority: P2)

購入希望者がカート画面でカートに入っている書籍の一覧を確認し、各書籍の数量を増減して
小計と合計金額がリアルタイムに更新されることを確認できる。

**Why this priority**: 書籍追加（P1）が完成した後に単独で機能するMVP拡張であり、
購入前の最終確認・調整の場として重要。

**Independent Test**: カートに書籍が入っている状態でカート画面を開き、
数量を変更して小計・合計が更新されることを確認することで単独テスト可能。

**Acceptance Scenarios**:

1. **Given** カートに複数の書籍が入っている状態で、**When** カート画面を開くと、**Then** 各書籍の書名・単価・数量・小計、および全書籍の合計金額が表示される
2. **Given** カート画面を表示している状態で、**When** ある書籍の数量を増やすと、**Then** その書籍の小計と合計金額がリアルタイムに更新される
3. **Given** カート画面を表示している状態で、**When** ある書籍の数量を減らすと、**Then** その書籍の小計と合計金額がリアルタイムに更新される
4. **Given** カートが空の状態で、**When** カート画面を開くと、**Then** カートが空である旨のメッセージが表示される

---

### User Story 3 - カートから書籍を削除する (Priority: P3)

購入希望者がカート画面で不要になった書籍の削除ボタンを押して、
カートから書籍を取り除き、合計金額が即時更新されることを確認できる。

**Why this priority**: P1・P2が完成した後に追加する機能。削除はカート操作の完成形であり、
数量調整（P2）だけでは「0冊にしたい」というニーズに対応できない。

**Independent Test**: カートに書籍がある状態でカート画面を開き、
削除ボタンを押してエントリが消え合計が更新されることで単独テスト可能。

**Acceptance Scenarios**:

1. **Given** カートに複数の書籍が入っている状態で、**When** 1冊の削除ボタンを押すと、**Then** その書籍がカートから除去され、合計金額が即時更新される
2. **Given** カートに1冊だけ書籍がある状態で、**When** その書籍の削除ボタンを押すと、**Then** カートが空になり、カードが空である旨のメッセージが表示される

---

### Edge Cases

- カートが空の状態でカート画面を開いた場合、「カートに商品がありません」等のメッセージを表示する
- 同一書籍を複数回「カートに追加」した場合、重複エントリを作成せず数量を加算する
- 数量を1未満にする操作（減算ボタン）は数量が1のときは実行できない（または削除として扱う）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ユーザーは書籍詳細画面からカートに書籍を追加できなければならない
- **FR-002**: すでにカートに存在する書籍を再度追加した場合、数量を加算しなければならない（重複エントリ禁止）
- **FR-003**: カート画面では書名・単価・数量・小計を書籍ごとに表示しなければならない
- **FR-004**: カート画面ではカート内全書籍の合計金額（送料除く）を表示しなければならない
- **FR-005**: ユーザーはカート内の書籍の数量を増減できなければならない
- **FR-006**: 数量変更時に小計と合計金額がリアルタイムに更新されなければならない
- **FR-007**: ユーザーはカートから任意の書籍を削除できなければならない
- **FR-008**: 書籍削除後に合計金額が即時更新されなければならない
- **FR-009**: カートが空の場合、空である旨のメッセージを表示しなければならない
- **FR-010**: 数量の下限は1とし、1未満への減算操作は受け付けない（または削除として処理する）

### Key Entities *(include if feature involves data)*

- **書籍 (Book)**: カートに追加される対象。書名・単価を持つ（参照のみ、カート機能では変更しない）
- **カート (Cart)**: セッション単位で管理される購入前の一時的な書籍の置き場。複数のカートアイテムを保持する
- **カートアイテム (CartItem)**: カート内の1エントリ。書籍・数量・小計の情報を持つ。小計は「単価 × 数量」で算出される

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 書籍詳細画面から「カートに追加」ボタン押下によるカート追加が1操作（1クリック）で完了する
- **SC-002**: カート内の数量変更後、小計と合計金額の更新が即時（体感遅延なし）で反映される
- **SC-003**: カートからの書籍削除後、合計金額の更新が即時（体感遅延なし）で反映される
- **SC-004**: 同一書籍のカート追加が100%のケースで数量加算として処理され、重複エントリが発生しない
- **SC-005**: カート画面の初期表示が3秒以内に完了する

## Assumptions

- カートの保持はセッション単位とし、ブラウザを閉じると消去される（永続化はスコープ外）
- 送料は合計金額に含めない（スコープ外）
- 在庫数チェックは行わない（在庫管理はスコープ外）
- 数量の下限は1とする（0にする場合は削除ボタンを使用する）
- 数量の上限は設定しない（在庫管理スコープ外のため）
- 書籍の価格は税込価格として表示する（税計算ロジックはスコープ外）
- 合計金額はカートアイテムの小計の合算とする（割引・クーポンはスコープ外）
