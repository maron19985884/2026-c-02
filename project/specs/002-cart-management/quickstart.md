# Quickstart: カート機能（追加・変更・削除）の動作確認

**Feature**: 002-cart-management
**Date**: 2026-06-28

## 前提条件

- Docker Compose が起動していること (`docker-compose up -d`)
- frontend: http://localhost:3000
- backend: http://localhost:4000
- books テーブルに2件以上のサンプルデータが登録されていること

## シナリオ 1: 書籍をカートに追加する

1. `http://localhost:3000/` を開いて書籍一覧を確認する
2. 任意の書籍カードをクリックして詳細画面（`/books/:id`）へ遷移する
3. 「カートに追加」ボタンをクリックする
4. 商品一覧（`/`）へ遷移すること
5. カート画面（`/cart`）を開く
6. 追加した書籍の書名・単価・数量（1）・小計が表示されること
7. 合計金額 = 単価 × 1 が表示されること

**期待結果**: ✅ カートに書籍が1件追加される

## シナリオ 2: 同一書籍の重複追加→数量加算

1. シナリオ 1 の状態から（書籍Aが1件カートにある）
2. 同じ書籍Aの詳細画面を開いて「カートに追加」をクリック
3. カート画面を開く
4. 書籍Aのエントリが1件のみ存在し、数量が 2 になっていること
5. 小計 = 単価 × 2 になっていること

**期待結果**: ✅ 重複エントリは作られず数量加算される

## シナリオ 3: 数量増減とリアルタイム合計更新

1. カート画面（`/cart`）を開く
2. ある書籍の数量増加ボタン（+）をクリックする
3. その書籍の小計と合計金額が即時更新されること（遅延なし）
4. 数量減少ボタン（−）をクリックする
5. 数量が 1 のとき、減少ボタンが非活性（disabled）になっていること

**期待結果**: ✅ 数量変更に即座に合計が追従する。数量下限 1 が守られる

## シナリオ 4: 書籍の削除と合計更新

1. カートに2種類の書籍が入っている状態でカート画面を開く
2. 1冊の削除ボタンをクリックする
3. その書籍がカートから除去され、合計金額が即時更新されること

**期待結果**: ✅ 削除後に合計が更新される

## シナリオ 5: 最後の1冊を削除→空カートメッセージ

1. カートに1種類の書籍のみ入っている状態でカート画面を開く
2. 削除ボタンをクリックする
3. カートが空になり「カートに商品がありません」等のメッセージが表示されること

**期待結果**: ✅ 空カートメッセージが表示される

## シナリオ 6: 複数書籍の同時管理

1. 書籍Aをカートに追加する
2. 書籍Bをカートに追加する
3. カート画面で2件のエントリと合計金額を確認する
4. 書籍Aの数量を2に変更する
5. 合計金額 = 書籍A単価×2 + 書籍B単価×1 になること

**期待結果**: ✅ 複数書籍の合計が正しく計算される
