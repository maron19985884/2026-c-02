# project.md — プロジェクト情報

> **記入担当:** プロジェクトリーダーまたはアーキテクト
> **更新タイミング:** 技術スタック・制約・メンバー変更時

---

## 1. プロジェクト概要

- **システム名:** 個人運営オンライン書店（購買フロー特化版）
- **目的:** 購入希望者が欲しい本を見つけて注文を完了できる、シンプルな購買フローを提供する
- **対象業務領域:** ECサイト（書籍販売）／購買フロー（商品閲覧〜注文完了）

---

## 2. 技術スタック

- **言語:** TypeScript
- **フレームワーク:** Next.js 14（フロントエンド）/ Express 4（バックエンド REST API）
- **DB / DWH:** MySQL 8.0
- **インフラ:** Docker Compose（frontend:3000 / backend:4000 / mysql:3306）
- **使用可能ライブラリ:**
  - フロントエンド: next, react 18, react-dom
  - バックエンド: express, cors, mysql2

---

## 3. 非機能要件

- **性能:** 画面遷移・API応答は3秒以内
- **可用性:** 開発・学習用途のため特段の稼働率要件なし
- **セキュリティ:** DB接続情報は `.env` で管理しコードに直書きしない

---

## 4. 制約条件

- ログイン・会員管理・決済処理・在庫管理・管理画面・レビュー・検索フィルターはスコープ外
- 送料計算はスコープ外（合計金額のみ表示）

---

## 5. 開発ルール（チーム固有）

- **ブランチ戦略:** main / feature/xxx
- **コミットメッセージ規則:** `feat:`, `fix:`, `chore:` プレフィックスを使用
- **レビュー人数:** 最低1名のApproveが必要
- **リンター / フォーマッター:** TypeScript 標準（tsc による型チェック）
