## 1. 背景・目的

個人が運営するオンライン書店において、購入希望者が書籍を探して注文を完了するまでの
購買フローをシンプルに実現する。ログインや決済といった複雑な機能は持たず、
「商品一覧 → 商品詳細 → カート → 注文フォーム → 注文完了」の5画面に絞る。

---

## 2. 対象業務・スコープ

### 対象
- 書籍一覧の閲覧
- 書籍詳細の閲覧
- カートへの追加・数量変更・削除
- 注文情報（氏名・住所・メール）の入力と注文確定
- 注文完了画面の表示

### 対象外
- ログイン・会員管理
- 決済処理
- 在庫管理
- 管理画面
- レビュー・評価
- 検索・フィルター
- 送料計算

---

## 3. 機能要件

| 要件ID | 要件名 | 内容 | 優先度 |
|---|---|---|---|
| REQ-001 | 書籍一覧表示 | 販売中の書籍をグリッド形式で一覧表示する。各書籍に書影・タイトル・著者・価格を表示する | 高 |
| REQ-002 | 書籍詳細表示 | 一覧から書籍をクリックすると詳細画面へ遷移し、書影・タイトル・著者・価格・説明文を表示する | 高 |
| REQ-003 | カート追加 | 詳細画面の「カートに追加」ボタンで書籍をカートに追加する。追加後は一覧画面に戻れる | 高 |
| REQ-004 | カート確認 | カート内の書名・単価・数量・小計・合計金額を一覧表示する | 高 |
| REQ-005 | カート数量変更 | カート内で数量を増減でき、小計と合計金額がリアルタイムに更新される | 高 |
| REQ-006 | カート削除 | カートから任意の書籍を削除でき、削除後に合計金額が即時更新される | 高 |
| REQ-007 | 注文フォーム入力 | 氏名・住所・メールアドレスを入力する（全項目必須）。入力時にカート内容と合計金額も同画面で確認できる | 高 |
| REQ-008 | バリデーション | 未入力・形式不正の場合はエラーメッセージを表示する | 高 |
| REQ-009 | 注文確定 | 「注文する」ボタン1クリックで注文を確定し注文完了画面へ遷移する | 高 |
| REQ-010 | 注文完了表示 | 注文完了メッセージと注文番号を表示する。商品一覧へ戻るリンクを設ける | 高 |

---

## 4. 非機能要件

| 区分 | 要件 |
|---|---|
| 性能 | 画面遷移・API応答は3秒以内 |
| 可用性 | 開発・学習用途のため特段の稼働率要件なし |
| セキュリティ | DB接続情報は環境変数で管理し、コードへの直書きを禁止する |
| 保守性 | フロント・バックエンドを分離し、APIを介してデータ連携する |

---

## 5. 制約条件

- 送料はスコープ外とし、合計金額のみ表示する
- ユーザー認証は実装しない（ゲスト購入のみ）
- 決済連携は実装しない

---

## 6. 用語定義

| 用語 | 定義 |
|---|---|
| カート | 購入前に書籍を一時的にためておく仮置き場。セッション単位で管理する |
| 注文番号 | 注文確定時にシステムが発行する一意の識別番号 |
| 小計 | 単価 × 数量の金額 |
| 合計金額 | カート内の全書籍の小計の合算（送料含まず） |
