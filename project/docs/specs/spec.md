## 1. 機能概要

個人運営オンライン書店の購買フロー全体を担う。以下5画面で構成される。

```
商品一覧 → 商品詳細 → （カートに追加）
                            ↓
                         カート → 注文フォーム → 注文完了
```

---

## 2. 画面別仕様

---

### 画面1｜商品一覧（`/`）

**表示内容**

| 項目 | 内容 |
|---|---|
| 書影 | 書籍のサムネイル画像 |
| タイトル | 書籍名 |
| 著者 | 著者名 |
| 価格 | 税込価格（円） |

**レイアウト:** グリッド形式（複数冊を同時に一覧表示）

**操作**
- 書籍カードをクリック → 商品詳細画面へ遷移（REQ-001, REQ-002）

---

### 画面2｜商品詳細（`/books/:id`）

**表示内容**

| 項目 | 内容 |
|---|---|
| 書影 | 書籍の拡大サムネイル画像 |
| タイトル | 書籍名 |
| 著者 | 著者名 |
| 価格 | 税込価格（円） |
| 説明文 | 書籍の紹介文 |

**操作**
- 「カートに追加」ボタン押下 → カートに追加し、商品一覧画面へ戻る（REQ-003）

---

### 画面3｜カート（`/cart`）

**表示内容**

| 項目 | 内容 |
|---|---|
| 書名 | カート内の書籍名 |
| 単価 | 税込価格（円） |
| 数量 | 増減可能なカウンター |
| 小計 | 単価 × 数量（リアルタイム更新） |
| 合計金額 | 全小計の合算（送料除く、リアルタイム更新） |

**操作**
- 数量増減 → 小計・合計金額をリアルタイム更新（REQ-005）
- 削除ボタン押下 → 該当書籍をカートから除去し合計金額を即時更新（REQ-006）
- 「注文手続きへ」ボタン押下 → 注文フォーム画面へ遷移（REQ-004）

---

### 画面4｜注文フォーム（`/order`）

**入力項目**

| 項目名 | 型 | 必須 | バリデーション |
|---|---|---|---|
| 氏名 | テキスト | ○ | 未入力不可 |
| 住所 | テキスト | ○ | 未入力不可 |
| メールアドレス | テキスト | ○ | 未入力不可・メール形式チェック |

**表示内容（入力フォームと同画面）**

| 項目 | 内容 |
|---|---|
| 注文商品一覧 | カート内の書名・単価・数量・小計 |
| 合計金額 | カート合計（送料除く） |

**操作**
- 「注文する」ボタン押下
  - バリデーション通過 → 注文確定処理 → 注文完了画面へ遷移（REQ-009）
  - バリデーション失敗 → 各項目にエラーメッセージを表示（REQ-008）

---

### 画面5｜注文完了（`/order/complete`）

**表示内容**

| 項目 | 内容 |
|---|---|
| 完了メッセージ | 「ご注文ありがとうございます」等の受付確認文言 |
| 注文番号 | システム発行の一意な番号 |
| 一覧リンク | 「商品一覧へ戻る」リンク（`/` へ遷移） |

## 3. 処理仕様

### 正常系フロー

1. ユーザーが商品一覧を閲覧する
2. 書籍カードをクリックして詳細画面へ遷移する
3. 「カートに追加」を押してカートに追加し、一覧へ戻る
4. カート画面で数量を確認・調整する
5. 「注文手続きへ」を押して注文フォームへ遷移する
6. 氏名・住所・メールアドレスを入力して「注文する」を押す
7. 注文完了画面で注文番号を確認する

### 異常系・例外フロー

| ケース | 対応 |
|---|---|
| 注文フォームで必須項目が未入力 | 該当フィールド下にエラーメッセージを表示し、注文処理は実行しない |
| メールアドレスの形式が不正 | 「メールアドレスの形式が正しくありません」と表示 |
| カートが空の状態で注文フォームへアクセス | カート画面へリダイレクト（<!-- 要確認: 空カート時の注文フォーム直接アクセス時の挙動 -->） |

---

## 4. 業務ルール

- カートの内容はセッション単位で保持する（永続化はスコープ外）
- 注文番号は注文確定時にバックエンドが採番する
- 送料は合計金額に含めない（スコープ外）
- 同一書籍をカートに追加した場合、数量を加算する

## 5. API仕様（フロントエンド ↔ バックエンド）

| メソッド | パス | 説明 |
|---|---|---|
| GET | `/api/books` | 書籍一覧を取得 |
| GET | `/api/books/:id` | 書籍詳細を取得 |
| POST | `/api/orders` | 注文を確定し注文番号を返す |
