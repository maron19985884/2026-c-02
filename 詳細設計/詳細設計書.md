## 3. データベース詳細設計

### 3.1 ER図

```
books ─────────────────────── order_items ──── orders
  id (PK)                       id (PK)          id (PK)
  title                         order_id (FK) ──▶ order_number
  author                        book_id           customer_name
  price                         title             customer_email
  description                   price             customer_address
  image_url                     quantity          total_amount
  created_at                                      created_at
  updated_at
```

### 3.2 テーブル定義

#### books テーブル

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | INT AUTO_INCREMENT | NOT NULL | - | 主キー |
| title | VARCHAR(255) | NOT NULL | - | 書名 |
| author | VARCHAR(255) | NOT NULL | - | 著者名 |
| price | INT | NOT NULL | - | 価格（税込・円） |
| description | TEXT | NULL | NULL | 書籍説明 |
| image_url | VARCHAR(500) | NULL | NULL | 書影URL |
| created_at | TIMESTAMP | NOT NULL | CURRENT_TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新日時 |

#### orders テーブル

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | INT AUTO_INCREMENT | NOT NULL | - | 主キー |
| order_number | VARCHAR(50) | NOT NULL | - | 注文番号（UNIQUE） |
| customer_name | VARCHAR(100) | NOT NULL | - | 氏名 |
| customer_email | VARCHAR(255) | NOT NULL | - | メールアドレス |
| customer_address | TEXT | NOT NULL | - | 住所 |
| total_amount | INT | NOT NULL | - | 合計金額（円） |
| created_at | TIMESTAMP | NOT NULL | CURRENT_TIMESTAMP | 注文日時 |

#### order_items テーブル

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | INT AUTO_INCREMENT | NOT NULL | - | 主キー |
| order_id | INT | NOT NULL | - | 外部キー（orders.id） |
| book_id | INT | NOT NULL | - | 書籍ID（参照のみ） |
| title | VARCHAR(255) | NOT NULL | - | 書名（注文時スナップショット） |
| price | INT | NOT NULL | - | 単価（注文時スナップショット） |
| quantity | INT | NOT NULL | 1 | 数量 |

> **スナップショット設計の理由**: 書籍の価格・タイトルが変更された場合でも注文当時の情報を保持するため、order_items に title・price を冗長保持する。

### 4.4 注文作成

| 項目 | 内容 |
|---|---|
| メソッド | POST |
| パス | /api/orders |
| Content-Type | application/json |
| 認証 | 不要 |

**リクエストボディ:**
```json
{
  "customer_name": "山田 太郎",
  "customer_email": "yamada@example.com",
  "customer_address": "東京都渋谷区〇〇 1-2-3",
  "items": [
    { "book_id": 1, "quantity": 2 },
    { "book_id": 3, "quantity": 1 }
  ]
}
```

**バリデーションルール:**
| フィールド | ルール |
|---|---|
| customer_name | 必須・空白のみ不可 |
| customer_email | 必須・正規表現 `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` |
| customer_address | 必須・空白のみ不可 |
| items | 必須・1件以上 |
| items[].book_id | DBに存在する書籍IDであること |
| items[].quantity | 1以上の整数（小数は切り捨て） |

**レスポンス例（201 Created）:**
```json
{
  "id": 42,
  "order_number": "ORD-20260504153000-1234",
  "total_amount": 10560
}
```

**エラーレスポンス:**
| ステータス | 内容 |
|---|---|
| 400 | バリデーションエラー（エラーメッセージ付き） |
| 500 | 書籍ID不正・DBエラー（トランザクションロールバック） |

**注文番号生成規則:** `ORD-YYYYMMDDHHmmss-XXXX`（XXXX は 1000〜9999 のランダム4桁）

## 5. 画面詳細設計

### 5.1 画面遷移図

```
[商品一覧 /]
    │
    │ 書籍カードをクリック
    ▼
[商品詳細 /books/:id]
    │
    │ 「カートに追加」ボタン押下
    │   → カートに追加（ローカル状態）
    │
    │ 「カートを見る」または Header の「カート」押下
    ▼
[カート /cart]
    │
    │ 「注文手続きへ進む」押下
    ▼
[注文フォーム /checkout]
    │
    │ 「注文する」ボタン押下
    │   → POST /api/orders
    │   → カートをクリア
    ▼
[注文完了 /complete/:orderId]
    │
    │ 「書籍一覧へ戻る」リンク
    ▼
[商品一覧 /]
```

### 5.2 各画面詳細

#### 画面1: 商品一覧 (`/`)

| 要件ID | 実装内容 |
|---|---|
| U-01 | `GET /api/books` で全書籍取得・グリッド表示 |
| U-02 | 各カードに書影（CSS生成）・タイトル・著者・価格を表示 |
| U-03 | カードクリックで `/books/:id` へ遷移 |

- **状態**: `books: Book[]`, `loading: boolean`, `error: string|null`
- **レンダリング**: `useEffect` で初回マウント時に API コール
- **書影**: タイトル先頭文字コードからカラーを算出する CSS グラデーション

#### 画面2: 商品詳細 (`/books/:id`)

| 要件ID | 実装内容 |
|---|---|
| U-04 | `GET /api/books/:id` で書籍詳細取得・全情報表示 |
| U-05 | 「カートに追加」ボタン → `CartContext.addToCart()` 呼び出し |
| U-06 | 追加後も画面遷移なし・2秒後にボタン状態リセット |

- **状態**: `book: Book|null`, `loading`, `error`, `added: boolean`（追加済みフラグ）
- **パラメータ**: `useParams().id` で書籍ID取得

#### 画面3: カート (`/cart`)

| 要件ID | 実装内容 |
|---|---|
| U-07 | `CartContext.items` から書名・単価・数量・小計を表示 |
| U-08 | ±ボタンで `updateQuantity()` → 0以下で自動削除・合計リアルタイム更新 |
| U-09 | 「削除」ボタンで `removeFromCart()` → 合計即時更新 |
| U-10 | サマリーカードに合計金額を常時表示 |
| U-11 | 「注文手続きへ進む」ボタンで `/checkout` へ遷移 |

- **カートが空の場合**: 空状態メッセージと一覧へのリンクを表示

#### 画面4: 注文フォーム (`/checkout`)

| 要件ID | 実装内容 |
|---|---|
| U-12 | 氏名・メールアドレス・住所の入力フィールド（全必須） |
| U-13 | `validate()` 関数でバリデーション・エラーメッセージをフィールド直下に表示 |
| U-14 | 右カラムに注文商品一覧と合計金額を常時表示 |
| U-15 | 「注文する」ボタンで `POST /api/orders` → カートクリア → `/complete/:id` へ遷移 |

- **バリデーションタイミング**: submit時 + 各フィールド変更時（エラークリア）
- **二重送信防止**: `submitting` フラグで送信中はボタン無効化

#### 画面5: 注文完了 (`/complete/:orderId`)

| 要件ID | 実装内容 |
|---|---|
| U-16 | 「ご注文ありがとうございます」メッセージ + チェックアイコン表示 |
| U-17 | `GET /api/orders/:id` で注文番号を取得して画面に表示 |
| U-18 | 「書籍一覧へ戻る」リンクで `/` へ遷移 |

- **表示内容**: 注文番号・注文日時・お届け先情報・注文明細・合計金額

## 6. カート設計

カートはサーバーを使わず、クライアントサイドで完結する設計とする。

```
CartContext（useReducer）
    │
    ├── ADD     : 書籍をカートに追加（同一書籍は数量+1）
    ├── UPDATE  : 数量変更（0以下は自動削除）
    ├── REMOVE  : 指定書籍を削除
    ├── CLEAR   : カート全削除（注文完了後に実行）
    └── HYDRATE : localStorage から初期状態を復元

localStorage キー: "bookstore_cart"
永続化タイミング: items 変化時（useEffect）
```

## 8. エラーハンドリング方針

| レイヤー | 方針 |
|---|---|
| Frontend API呼び出し | try/catch でエラーを捕捉し、画面にエラーメッセージ表示 |
| Backend ルーター | try/catch で捕捉、適切な HTTP ステータスコードとエラーメッセージを返却 |
| 注文作成 | トランザクションを使用し、途中失敗時はロールバック |
| バリデーション | Frontend と Backend の両方でバリデーションを実施（多重防御） |
