# CLAUDE.md — オンライン書店プロジェクト

## プロジェクト概要

個人運営オンライン書店の購買フローWebアプリ。  
要件定義書（user_requirements.md）に基づき、商品一覧 → 商品詳細 → カート → 注文フォーム → 注文完了 の5画面を実装する。

---

## 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | React + TypeScript + Vite |
| バックエンド | Node.js + TypeScript + Express（REST API） |
| DB | MySQL 8.0（docker-compose経由） |
| コンテナ | Docker（~/docker-compose.yml を使用） |

## 開発ルール

### コーディング規約

- TypeScriptの`strict`モードを有効にすること
- `any`型の使用を禁止。型が不明な場合は`unknown`を使用し適切にナローイングすること
- コンポーネントはすべて関数コンポーネント（Hooks使用）で実装すること
- APIレスポンス型はフロントエンド・バックエンドで共通のinterfaceを定義すること（`types/`配下）
- エラーハンドリングは必ずtry-catchで実装し、適切なHTTPステータスを返すこと

### API設計原則

- RESTful設計に従うこと
- レスポンスは常に `{ data, error, message }` の統一フォーマットとすること
- バリデーションエラーは HTTP 400、サーバーエラーは HTTP 500 を返すこと

### DB接続

- ホームディレクトリの `docker-compose.yml` で起動中のMySQLコンテナに接続すること
- 接続情報は `.env` で管理し、`mysql2/promise` を使用すること
- SQLインジェクション対策としてプリペアドステートメントを必ず使用すること

## 実装対象画面と要件

| 画面 | 要件ID | 実装内容 |
|---|---|---|
| 商品一覧 | U-01〜U-03 | グリッド表示、書影/タイトル/著者/価格、詳細画面への遷移 |
| 商品詳細 | U-04〜U-06 | 詳細情報表示、カート追加、一覧への戻り |
| カート | U-07〜U-11 | 商品一覧確認、数量変更、削除、合計表示、注文フォームへ遷移 |
| 注文フォーム | U-12〜U-15 | 氏名/住所/メール入力、バリデーション、カート確認表示、注文確定 |
| 注文完了 | U-16〜U-18 | 完了メッセージ、注文番号表示、一覧への戻りリンク |

---

## スコープ外（実装禁止）

- ログイン・会員管理
- 決済処理
- 在庫管理
- 管理画面
- レビュー・評価機能
- 検索・フィルター機能

---

## 実装手順

1. `docker-compose.yml` のMySQL設定を確認し `.env` を作成
2. `db/init.sql` でテーブル作成・サンプルデータ投入
3. バックエンドのAPIエンドポイントを実装
4. フロントエンドのページ・コンポーネントを実装
5. フロントエンド・バックエンドの疎通確認
6. 単体テスト実施

---

## 禁止事項

- `console.log` をコミットに含めないこと（デバッグ用途のみ使用し必ず削除すること）
- ハードコードされた接続情報（DB password等）をソースに含めないこと
- スコープ外機能の実装を独自判断で追加しないこと
