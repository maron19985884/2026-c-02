#### 2. ユニットテスト（51件）

**テストフレームワーク:** Jest 29 + ts-jest  
**実行環境:** node:20-alpine（Docker コンテナ）  
**外部依存:** なし（純粋な関数ロジックのみテスト）

| テストファイル | 対象ソース | テスト件数 |
|-------------|---------|---------|
| `backend/generateOrderNumber.test.ts` | `backend/src/routes/orders.ts` | 4件 |
| `backend/validation.test.ts` | `backend/src/routes/orders.ts` | 20件 |
| `frontend/cartReducer.test.ts` | `frontend/src/context/CartContext.tsx` | 17件 |
| `frontend/checkoutValidation.test.ts` | `frontend/src/app/checkout/page.tsx` | 10件 |

**テスト項目概要:**

- 注文番号生成: フォーマット検証（`ORD-YYYYMMDDHHmmss-XXXX`）、範囲検証、一意性検証
- バリデーション: 氏名・メールアドレス・住所・注文商品の必須/形式チェック
- カートReducer: ADD / UPDATE / REMOVE / CLEAR / HYDRATE の各アクション、イミュータブル性、合計金額・数量計算
- チェックアウトフォーム: 各フィールドの必須/形式バリデーション、複合エラー

**テスト結果:**

```
Test Suites: 4 passed, 4 total
Tests:       51 passed, 51 total
Time:        10.656 s
```

#### 3. 結合テスト（16件）

**テストフレームワーク:** Jest 29 + ts-jest + Axios  
**実行環境:** node:20-alpine（Docker コンテナ）、`2026-c-02_app-network` に接続  
**接続先:** `http://backend:4000`（Docker 内部）

| テストファイル | 対象 API | テスト件数 |
|-------------|---------|---------|
| `api/books.test.ts` | GET /api/books、GET /api/books/:id | 5件 |
| `api/orders.test.ts` | POST /api/orders、GET /api/orders/:id | 11件 |

**テスト項目概要:**

- `GET /api/books`: 200レスポンス・レスポンス構造検証
- `GET /api/books/:id`: 正常取得・404・400 の各ケース
- `POST /api/orders`: 注文作成成功（201）・各バリデーションエラー（400）・不正書籍ID（500）
- `GET /api/orders/:id`: 注文詳細取得・注文明細件数・合計金額の正確性・404・400

**テスト結果:**

```
Test Suites: 2 passed, 2 total
Tests:       16 passed, 16 total
Time:        11.628 s
```

#### 4. テスト実行方法

**ユニットテスト（Docker Desktop 起動後）:**
```bash
docker run --rm \
  -v "d:/IBMStudy/2026-c-02/テスト/ユニットテスト:/tests" \
  -w /tests node:20-alpine \
  sh -c "npm install && npm test"
```

**結合テスト（`docker-compose up -d` 後）:**
```bash
docker run --rm \
  -v "d:/IBMStudy/2026-c-02/テスト/結合テスト:/tests" \
  -w /tests \
  --network 2026-c-02_app-network \
  -e API_URL=http://backend:4000 \
  node:20-alpine \
  sh -c "npm install && npm test"
```
