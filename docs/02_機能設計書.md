## 1. 画面機能詳細

### 1-1. 書籍一覧画面（/）

#### 概要

販売中の書籍をグリッド形式で一覧表示する。

#### 機能仕様

| # | 機能 | 要件ID | 詳細 |
|---|---|---|---|
| 1 | 書籍グリッド表示 | U-01 | `auto-fill / minmax(200px, 1fr)` による レスポンシブグリッド |
| 2 | 書籍情報表示 | U-02 | 書影・タイトル・著者・価格を各カードに表示 |
| 3 | 詳細遷移 | U-03 | カードクリックで `/books/[id]` へ遷移 |

#### コンポーネント構成

```
BookListPage (Server Component)
└── Header (Client Component)
│   └── カートアイコン（カート件数バッジ付き）
└── BookCard × N (Client Component)
    ├── 書影 (img)
    ├── タイトル
    ├── 著者
    └── 価格
```

#### データ取得

- `GET /api/books` を Server Component 内で fetch
- `revalidate = 60`（60秒 ISR）

---

### 1-2. 書籍詳細画面（/books/[id]）

#### 概要

選択した書籍の詳細情報を表示し、カートに追加できる。

#### 機能仕様

| # | 機能 | 要件ID | 詳細 |
|---|---|---|---|
| 1 | 詳細情報表示 | U-04 | 書影・タイトル・著者・価格・説明文を表示 |
| 2 | カート追加 | U-05 | 「カートに追加」ボタン押下で localStorage のカートに追加。同一商品は数量加算 |
| 3 | 一覧戻り | U-06 | 「← 書籍一覧に戻る」リンクで `/` へ遷移 |

#### コンポーネント構成

```
BookDetailPage (Client Component)
├── Header
├── ← 書籍一覧に戻る (Link)
├── 書影 (img)
└── 商品情報エリア
    ├── タイトル / 著者 / 価格 / 説明文
    ├── カートに追加ボタン（追加後2秒間 ✓ 表示）
    └── カートを見るボタン
```

#### 状態管理

| 状態 | 型 | 初期値 | 説明 |
|---|---|---|---|
| book | `Book \| null` | null | API レスポンス |
| error | string | '' | エラーメッセージ |
| added | boolean | false | カート追加完了フラグ |

### 1-3. カート画面（/cart）

#### 概要

カート内の商品を確認・編集し、注文フォームへ進む。

#### 機能仕様

| # | 機能 | 要件ID | 詳細 |
|---|---|---|---|
| 1 | 商品一覧確認 | U-07 | 書影・書名・単価・数量・小計を表示 |
| 2 | 数量変更 | U-08 | ＋/− ボタンで数量変更。小計・合計はリアルタイム再計算 |
| 3 | 削除 | U-09 | ✕ ボタンで商品削除。合計金額も即時更新 |
| 4 | 合計表示 | U-10 | 全商品の合計金額を画面下部に表示 |
| 5 | 注文フォームへ遷移 | U-11 | 「注文手続きへ」ボタンで `/checkout` へ遷移 |
| 6 | 空カート表示 | - | カートが空のときは空メッセージと一覧リンクを表示 |

#### カート状態管理（CartContext）

```typescript
interface CartContextValue {
  items: CartItem[];           // カート内商品リスト
  addItem: (book) => void;     // 商品追加（同一商品は quantity + 1）
  updateQuantity: (id, qty) => void;  // 数量変更
  removeItem: (id) => void;    // 商品削除
  clearCart: () => void;       // カートリセット（注文確定後）
  totalAmount: number;         // 合計金額（自動計算）
}
```

- `localStorage` キー: `bookstore_cart`
- マウント時に localStorage からカート状態を復元

### 1-4. 注文フォーム画面（/checkout）

#### 概要

顧客情報を入力して注文を確定する。

#### 機能仕様

| # | 機能 | 要件ID | 詳細 |
|---|---|---|---|
| 1 | 顧客情報入力 | U-12 | 氏名・住所・メールアドレス（全必須） |
| 2 | バリデーション | U-13 | 未入力・メール形式不正でフィールド下にエラーメッセージ表示 |
| 3 | 注文内容確認 | U-14 | 同画面右カラムに商品名・数量・小計・合計を表示 |
| 4 | 注文確定 | U-15 | 「注文する」ボタンで `POST /api/orders`。成功後 `/orders/:id` へ遷移 |

#### バリデーション仕様

| フィールド | 必須 | 形式チェック | エラーメッセージ |
|---|---|---|---|
| 氏名 | ○ | - | 「氏名を入力してください」 |
| 住所 | ○ | - | 「住所を入力してください」 |
| メールアドレス | ○ | RFC準拠正規表現 | 「メールアドレスを入力してください」/ 「メールアドレスの形式が正しくありません」 |

#### レイアウト

```
┌─────────────────────┬──────────────────────┐
│  顧客情報フォーム    │   注文内容確認        │
│                     │                       │
│  氏名 [____]        │  書籍名 × 数量  小計  │
│  住所 [____]        │  書籍名 × 数量  小計  │
│  メール [____]      │  ─────────────────    │
│                     │  合計  ¥XX,XXX        │
│  [注文する]         │                       │
└─────────────────────┴──────────────────────┘
```

## 2. API 詳細設計

### レスポンス共通フォーマット

```typescript
interface ApiResponse<T> {
  data: T | null;      // 正常時のデータ
  error: string | null; // エラー時のメッセージ
  message: string;      // 処理結果メッセージ
}
```

### GET /api/books

**概要:** 全書籍一覧を返す

**レスポンス例（200）:**
```json
{
  "data": [
    {
      "id": 1,
      "title": "JavaScript 第7版",
      "author": "David Flanagan",
      "price": 4840,
      "description": "...",
      "image_url": "https://...",
      "created_at": "2026-05-12T02:05:35.000Z",
      "updated_at": "2026-05-12T02:05:35.000Z"
    }
  ],
  "error": null,
  "message": "OK"
}
```

---

### GET /api/books/:id

**概要:** 指定IDの書籍詳細を返す

| ステータス | 条件 |
|---|---|
| 200 | 正常取得 |
| 400 | id が整数でない（例: /api/books/abc） |
| 404 | 指定IDの書籍が存在しない |
| 500 | サーバーエラー |

---

### POST /api/orders

**概要:** 注文を登録する（トランザクション処理）

**リクエストボディ:**
```json
{
  "customer_name": "山田 太郎",
  "customer_address": "東京都渋谷区1-1-1",
  "customer_email": "yamada@example.com",
  "items": [
    { "book_id": 1, "quantity": 2, "unit_price": 4840 },
    { "book_id": 3, "quantity": 1, "unit_price": 4180 }
  ]
}
```

**処理フロー:**

```
1. リクエストバリデーション
   ├── customer_name / address / email が空でないこと
   ├── email が正規表現に一致すること
   └── items が空でないこと
2. トランザクション開始
3. orders テーブルに INSERT
4. order_items テーブルに INSERT（items 分ループ）
5. トランザクションコミット
6. 作成した orders レコードを返す
※ エラー時はロールバック
```

| ステータス | 条件 |
|---|---|
| 201 | 注文作成成功 |
| 400 | バリデーションエラー |
| 500 | DB エラー |

---

### GET /api/orders/:id

**概要:** 注文詳細（注文明細・書籍名含む）を返す

**レスポンス例（200）:**
```json
{
  "data": {
    "id": 1,
    "customer_name": "山田 太郎",
    "customer_address": "東京都渋谷区1-1-1",
    "customer_email": "yamada@example.com",
    "total_amount": 13860,
    "created_at": "2026-05-12T02:09:07.000Z",
    "items": [
      { "book_id": 1, "quantity": 2, "unit_price": 4840, "title": "JavaScript 第7版" },
      { "book_id": 3, "quantity": 1, "unit_price": 4180, "title": "Clean Architecture" }
    ]
  },
  "error": null,
  "message": "OK"
}
```
