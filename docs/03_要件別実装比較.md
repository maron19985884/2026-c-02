### U-13　入力ミスがあれば気づけること（エラーメッセージ表示）

| | スギワラ版 | コバヤシ版 | サトウ版 |
|-|-----------|-----------|---------|
| 達成度 | **○** | **△** | **△** |
| バリデーション実行タイミング | onBlur（欄を離れたとき）＋ onSubmit | onSubmit のみ | onSubmit のみ |
| 入力中のエラークリア | ○（onChange でクリア） | ×（送信後まで残る） | △（変更した項目のみクリア） |
| 未入力チェック | ○ | ○ | ○ |
| メール形式チェック | ○（`/^[^\s@]+@[^\s@]+\.[^\s@]+$/`） | ○（同正規表現） | ○（同正規表現） |
| 文字数チェック | ○（氏名100文字・住所255文字上限） | × | × |

**バリデーションタイミングの実装差異：**
```tsx
// スギワラ版 — onBlur で該当フィールドだけチェック
<input
  onBlur={() => {
    const err = validateField('customer_name', formData.customer_name);
    setErrors(prev => ({ ...prev, customer_name: err }));
  }}
  onChange={() => setErrors(prev => ({ ...prev, customer_name: '' }))}
/>

// コバヤシ版・サトウ版 — 送信ボタンを押したときにまとめてチェック
const handleSubmit = async (e) => {
  e.preventDefault();
  const errors = validate(formData);   // 全フィールドを一括チェック
  if (Object.keys(errors).length > 0) {
    setErrors(errors);
    return;
  }
  // → API 呼び出しへ
};
```

> **スギワラ版が○の理由：** 入力欄からフォーカスが外れた時点でそのフィールドのエラーを表示するため、送信前に間違いに気づき修正できる。コバヤシ版・サトウ版は送信ボタンを押して初めてエラーがわかる。
