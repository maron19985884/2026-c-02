# 外部設計書（機能仕様書）
## 画面3：カート

**対応要件:** U-07, U-08, U-09, U-10, U-11
**画面パス:** `/cart`
**Nextコンポーネント:** `src/app/cart/page.tsx`（クライアントコンポーネント）

---

## 1. 画面概要

カートに追加された書籍の一覧を表示し、数量変更・削除・合計確認・注文手続きへの進行を行う画面。データはサーバーとのAPI通信を行わず、`CartContext`（`localStorage` バックアップ）のみで管理する。

## 3. 表示項目と表示条件

### 3-1. ヘッダー（共通）

| 要素 | 表示内容 | 表示条件 |
|---|---|---|
| ロゴ/サイト名 | 書店名テキスト（リンク：`/`） | 常時表示 |
| カートアイコン | カートに入っている総冊数 | 常時表示 |

### 3-2. カートアイテム行（`CartItemRow` コンポーネント）

| 要素 | 表示内容 | 表示条件 |
|---|---|---|
| 書影（小） | `Book.coverImageUrl` の画像（サムネイル） | カートアイテムが1件以上あるとき |
| タイトル | `Book.title` | 同上 |
| 著者名 | `Book.author` | 同上 |
| 単価 | `¥{Book.price.toLocaleString()}` | 同上 |
| 数量表示 | 現在の `CartItem.quantity` | 同上 |
| 「−」ボタン | 数量を1減らす | `quantity ≥ 2` のとき活性。`quantity = 1` のとき非活性（グレーアウト） |
| 「+」ボタン | 数量を1増やす | 常時活性 |
| 小計 | `¥{(Book.price × CartItem.quantity).toLocaleString()}` | 同上。数量変更時にリアルタイム更新 |
| 「削除」ボタン | アイテムをカートから削除 | 常時表示 |

### 3-3. 合計金額エリア

| 要素 | 表示内容 | 表示条件 |
|---|---|---|
| 合計金額 | `合計：¥{totalAmount.toLocaleString()}` | カートアイテムが1件以上あるとき |
| 合計の補足 | 「※ 送料は含まれていません」等の文言は表示しない（スコープ外） | — |

### 3-4. ボタン類

| 要素 | 表示内容 | 表示条件 |
|---|---|---|
| 「注文手続きへ進む」ボタン | — | カートアイテムが1件以上あるとき |
| 「商品一覧へ戻る」ボタン | — | カートが空のとき |

## 4. ユーザー操作と動作

| 操作対象 | 操作 | 動作 |
|---|---|---|
| 「−」ボタン | クリック | `CartContext.updateQuantity(bookId, quantity - 1)` を呼ぶ。小計・合計をリアルタイム更新。`quantity = 1` のとき非活性 |
| 「+」ボタン | クリック | `CartContext.updateQuantity(bookId, quantity + 1)` を呼ぶ。小計・合計をリアルタイム更新 |
| 「削除」ボタン | クリック | `CartContext.remove(bookId)` を呼ぶ。対象アイテムが一覧から消え、合計金額即時更新。全件削除後は空カート表示に切り替わる |
| 「注文手続きへ進む」ボタン | クリック | 注文フォーム画面（`/checkout`）へ遷移 |
| 「商品一覧へ戻る」ボタン（空カート時） | クリック | 商品一覧画面（`/`）へ遷移 |
| ロゴ/サイト名（ヘッダー） | クリック | 商品一覧画面（`/`）へ遷移 |

**数量変更のリアルタイム更新仕様:**
- `updateQuantity` 呼び出し → `CartContext` のstate更新 → `localStorage` 保存 → 画面再レンダリング（Reactの状態更新に依存、明示的なAPI通信なし）
- 小計・合計は `useMemo` 等で `cartItems` の変化を検知して再計算

---

## 5. バリデーションルール

なし（APIコールがないため、クライアント側の状態操作のみ）

---

## 6. 使用するAPIエンドポイント

なし。カート状態は `CartContext`（`localStorage`）で管理するため、この画面でのAPIコールは発生しない。

> **補足:** カートアイテムの書籍情報（タイトル・著者・価格・書影）は `localStorage` に保持するか、`bookId` のみを保持して `CartContext` 初期化時に `GET /api/books` で補完する実装を選択する。後者の場合、ページロード時に1回だけAPIコールが発生する。

---

## 7. 画面遷移先

| 遷移先 | 条件 |
|---|---|
| 注文フォーム画面 `/checkout` | 「注文手続きへ進む」ボタンをクリック（カートに1件以上ある場合） |
| 商品一覧画面 `/` | 「商品一覧へ戻る」ボタンをクリック（空カート時）、またはヘッダーロゴクリック |
