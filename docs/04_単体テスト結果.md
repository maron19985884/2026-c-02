## 5. テスト方針

### テスト種別

本プロジェクトでは **結合テスト（Integration Test）** をメインに実施。  
supertest により Express アプリを直接起動し、HTTP レベルで API の振る舞いを検証する。

### テストデータ

- 実際の MySQL コンテナ（appdb）に接続して実行
- テストデータ（注文）は各テスト内で動的に作成
- テスト間のデータ隔離はせず、テスト実行後にロールバックしない方針（参照系テストはサンプルデータを使用）

### カバレッジ対象

| 対象 | 内容 |
|---|---|
| 正常系 | 全エンドポイントの正常レスポンス |
| 異常系（クライアントエラー） | バリデーション失敗（400）・リソース未存在（404） |
| DB 接続 | ヘルスチェックエンドポイントで DB 疎通を確認 |
| 計算ロジック | 合計金額の正確性（単品・複数商品） |
