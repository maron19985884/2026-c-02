---

## 1. システム概要

個人運営オンライン書店の購買フローに特化した Web アプリケーション。  
「書籍を見つける → カートに追加 → 注文情報を入力 → 注文完了」の一連のフローを提供する。

### 対象ユーザー

書店サイトを訪れた購入希望者（ログイン不要）

## 6. データフロー

### 購買フロー

```
[書籍一覧] ─→ [書籍詳細]
                  │ addItem()
                  ▼
             [カート（localStorage）]
                  │ checkout
                  ▼
             [注文フォーム] ─→ POST /api/orders
                                    │
                                    ▼
                              [注文完了] ─→ GET /api/orders/:id
```

### カート管理方針

- カートはサーバー非保存、ブラウザの `localStorage` に JSON として永続化
- セッション間でカートが保持される
- 注文確定後に `clearCart()` でリセット

## 8. 起動手順

```bash
# 全サービス起動（初回 or イメージ変更時）
docker compose up --build -d

# 通常起動
docker compose up -d

# 停止
docker compose down

# ローカル開発（Docker なし）
# ターミナル1: バックエンド
cd backend && npm run dev

# ターミナル2: フロントエンド
cd frontend && npm run dev
```

## 9. スコープ外機能

以下の機能は本システムに含まない。

- ログイン・会員管理
- 決済処理
- 在庫管理
- 管理画面
- レビュー・評価機能
- 検索・フィルター機能
