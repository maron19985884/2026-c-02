# 商品一覧表示機能 外部設計書

## 1. 概要
- **機能ID**: F-01
- **機能名**: 商品一覧表示機能
- **対象ユーザー**: 書店サイトを訪れた購入希望者（未ログインの一般訪問者）
- **目的・背景**: 販売中の書籍をまとめて閲覧し、興味のある書籍を選択して詳細画面へ遷移できるようにする。購買フローの起点となる画面であり、購入意欲を喚起することを目的とする。
- **作成日**: 2026-05-05
- **バージョン**: 1.0
- **関連要件**: U-01, U-02, U-03

## 2. 機能概要

ユーザーがサイトのトップページ（`/`）にアクセスすると、データベースに登録された全書籍の一覧をグリッド形式で表示する。各書籍カードには書影・タイトル・著者・価格の基本情報が表示され、カード（または書籍タイトル）をクリックすると当該書籍の詳細画面（`/products/[id]`）に遷移する。

本機能は購買フローのエントリーポイントであり、ユーザーが商品を発見・比較・選択するための基盤となる。

## 3. 画面・UI設計

### 3.1 画面一覧

| 画面名 | URL/パス | 説明 |
|---|---|---|
| 商品一覧画面 | `/` | 販売中の全書籍をグリッド形式で表示するトップページ |

### 3.2 画面レイアウト

#### ヘッダー領域
- サイトタイトル（例: 「オンライン書店」）
- カートアイコン／カートへのリンク（カート画面 `/cart` への導線）

#### メインコンテンツ
- 見出し（例: 「販売中の書籍一覧」）
- 書籍カードグリッド
  - レスポンシブグリッドレイアウト（PC: 3〜4列、タブレット: 2列、モバイル: 1列）
  - カードはホバーで影が深くなるなどの視覚フィードバック

#### 書籍カード構成要素

| 要素 | 内容 | スタイル要件 |
|---|---|---|
| 書影 | `image_url` の画像 | 上部に配置、固定サイズ（例: 200x280px） |
| タイトル | `title` | 太字、見出しサイズ |
| 著者 | `author` | サブテキスト（`#6b7280`） |
| 価格 | `¥{price.toLocaleString()}` | 強調表示、円マーク付き |

#### スタイル仕様（プロジェクト規約準拠）
- インラインスタイル（`style={{...}}`）のみ使用
- ページ背景: `#f3f4f6`
- カード背景: `white`、`box-shadow: 0 2px 8px rgba(0,0,0,0.08)`
- 本文テキスト: `#111827` / `#374151`
- アクセントカラー: `#1e3a8a` / `#1e40af`

### 3.3 画面遷移

```
[商品一覧 (/)]
     │
     ├─ 書籍カードクリック → [商品詳細 (/products/[id])]
     │
     └─ カートアイコンクリック → [カート (/cart)]
```

## 4. 入出力定義

### 4.1 入力項目

本画面はユーザーからの入力を受け付けない（閲覧専用）。ただし以下の操作を受け付ける：

| 操作 | 内容 | 結果 |
|---|---|---|
| 書籍カードクリック | 任意の書籍カードを選択 | `/products/[id]` へ遷移 |
| カートリンククリック | ヘッダーのカートリンク選択 | `/cart` へ遷移 |

### 4.2 出力項目（書籍カード単位）

| 項目名 | 型 | 説明 |
|---|---|---|
| id | number | 書籍ID（遷移先URLパラメータに使用） |
| title | string | 書籍タイトル |
| author | string | 著者名 |
| price | number | 価格（円、整数） |
| image_url | string | 書影画像URL |

`description` は一覧画面では表示しない。

## 5. API設計

### 5.1 使用エンドポイント

| メソッド | パス | リクエスト | レスポンス | 説明 |
|---|---|---|---|---|
| GET | `/products` | なし | `Book[]` | 書籍一覧を取得 |

#### レスポンス例

```json
[
  {
    "id": 1,
    "title": "プログラミング入門",
    "author": "山田 太郎",
    "price": 2800,
    "description": "プログラミングの基礎から...",
    "image_url": "https://placehold.co/200x280/4A90E2/ffffff?text=Programming",
    "created_at": "2026-05-05T00:00:00.000Z"
  }
]
```

#### 呼び出し方法
- 実行タイミング: ページマウント時（`useEffect` 内で1回）
- ベースURL: `process.env.NEXT_PUBLIC_API_URL`（デフォルト `http://localhost:4000`）
- 認証: 不要

### 5.2 新規エンドポイント

新規エンドポイントは不要（既存の `GET /products` を使用）。

## 6. データ設計

### 6.1 使用テーブル

#### books テーブル

| カラム名 | 型 | 用途 |
|---|---|---|
| id | INT (PK) | 一意識別子 |
| title | VARCHAR(200) | 表示 |
| author | VARCHAR(100) | 表示 |
| price | INT | 表示 |
| description | TEXT | 取得するが画面では未使用 |
| image_url | VARCHAR(500) | 表示 |
| created_at | TIMESTAMP | 未使用 |

並び順: `ORDER BY id`（登録順）。

### 6.2 データフロー

```
[MySQL: books] → [Backend: GET /products] → [Frontend: page.tsx (useEffect)]
                                                      ↓
                                              [グリッド表示]
```

1. ユーザーが `/` にアクセス
2. クライアント側で `GET /products` を実行
3. 取得した `Book[]` を state にセット
4. グリッド形式でカードをレンダリング

## 7. ビジネスロジック

- 書籍は `books` テーブルに存在する全件を表示する（在庫管理はスコープ外のため、削除・非表示の概念なし）。
- 並び順は ID 昇順（登録順）。
- 1冊もデータがない場合は「販売中の書籍はありません」等の空状態メッセージを表示する。
- ローディング中（API レスポンス待ち）は「読み込み中...」等のインジケータを表示する。

## 8. エラー処理

| エラー条件 | エラーメッセージ | 対処方法 |
|---|---|---|
| API通信失敗（ネットワークエラー） | 「書籍一覧の取得に失敗しました。時間をおいて再度お試しください。」 | エラーメッセージ表示、リトライボタンを提供（任意） |
| サーバーエラー（500） | 同上 | 同上 |
| データ0件 | 「販売中の書籍はありません」 | 空状態を明示的に表示 |
| 画像読み込み失敗 | （個別エラー） | `onError` で代替画像／プレースホルダーを表示 |

## 9. 非機能要件

- **パフォーマンス**:
  - 初回表示: 2秒以内（API応答 + レンダリング）
  - 画像は外部CDN（placehold.co等）からの読み込みで、必要に応じて遅延読み込み（`loading="lazy"`）を検討
- **セキュリティ**:
  - 認証不要の公開エンドポイントを使用
  - XSS対策: React のデフォルト挙動（テキストノードの自動エスケープ）に依拠
  - 画像URLは信頼できるソースのみを許容（DB登録時に管理者が設定する想定）
- **アクセシビリティ**:
  - 書影には `alt` 属性（タイトル）を設定
  - カードはリンク化し、キーボード操作（Tab/Enter）で遷移可能にする
  - カラーコントラスト比 WCAG AA 準拠

## 10. 制約・前提条件

- 検索・フィルター機能はスコープ外（要件書 3章）。
- ページング機能は要件にないため未実装（書籍数が少ない前提）。
- 在庫管理はスコープ外のため、表示中の全書籍が常に注文可能とみなす。
- ログイン機能なしのため、誰でもアクセス可能。
- フロントエンドコンポーネントは `"use client"` ディレクティブが必須。
- スタイルはインラインスタイルのみ使用（CSSモジュール・Tailwind不可）。
