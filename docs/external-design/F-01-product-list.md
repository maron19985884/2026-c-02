# 商品一覧表示機能 外部設計書

## 1. 概要
- **機能ID**: F-01
- **機能名**: 商品一覧表示機能
- **対象ユーザー**: 書店サイトを訪れた購入希望者（未ログインの一般訪問者）
- **目的・背景**: 販売中の書籍をまとめて閲覧し、興味のある書籍を選択して詳細画面へ遷移できるようにする。購買フローの起点となる画面であり、購入意欲を喚起することを目的とする。
- **作成日**: 2026-05-05
- **バージョン**: 1.1
- **関連要件**: U-01, U-02, U-03

## 2. 機能概要

ユーザーがサイトのトップページ（`/`）にアクセスすると、データベースに登録された全書籍の一覧をグリッド形式で表示する。各書籍カードには書影・タイトル・著者・価格・星評価・Prime情報が表示され、カードクリックで当該書籍の詳細画面（`/products/[id]`）へ遷移する。カード内の「カートに追加」ボタンで、詳細画面に遷移せず直接カートに追加することも可能。

### 3.3 画面遷移

```
[商品一覧 (/)]
     │
     ├─ 書籍カードクリック（画像/タイトル） → [商品詳細 (/products/[id])]
     ├─ 「カートに追加」ボタンクリック      → CartContext.addItem()（同画面）
     └─ カートアイコンクリック              → [カート (/cart)]
```

## 4. 入出力定義

### 4.1 入力項目

本画面はユーザーからの入力を受け付けない（閲覧・操作専用）。受け付ける操作：

| 操作 | 内容 | 結果 |
|---|---|---|
| 書籍カードクリック（画像/タイトル） | 任意の書籍カードのリンク領域を選択 | `/products/[id]` へ遷移 |
| 「カートに追加」ボタンクリック | カード内ボタン押下 | `addItem({ id, title, author, price })` を呼び出し |
| カートリンククリック | ヘッダーのカートアイコン選択 | `/cart` へ遷移 |

### 4.2 出力項目（書籍カード単位）

| 項目名 | 型 | 説明 |
|---|---|---|
| id | number | 書籍ID（遷移先URLパラメータ・カート追加に使用） |
| title | string | 書籍タイトル |
| author | string | 著者名 |
| price | number | 価格（円、整数） |
| image_url | string | 書影画像URL |

`description` は一覧画面では表示しない。

#### ダミー評価の算出ロジック
```typescript
const stars = Math.min(5, parseFloat((3.5 + (id % 5) * 0.3).toFixed(1)));
const count = 100 + (id * 137) % 900;
```
DB の評価機能はスコープ外のため、ID から決定的に生成する。

## 5. API設計

### 5.1 使用エンドポイント

| メソッド | パス | リクエスト | レスポンス | 説明 |
|---|---|---|---|---|
| GET | `/products` | なし | `Book[]` | 書籍一覧を取得 |

#### レスポンス例

```json
[
  {
    "id": 1,
    "title": "プログラミング入門",
    "author": "山田 太郎",
    "price": 2800,
    "description": "プログラミングの基礎から...",
    "image_url": "https://placehold.co/200x280/4A90E2/ffffff?text=Programming",
    "created_at": "2026-05-05T00:00:00.000Z"
  }
]
```

#### 呼び出し方法
- 実行タイミング: ページマウント時（`useEffect` 内で1回）
- ベースURL: `process.env.NEXT_PUBLIC_API_URL`（デフォルト `http://localhost:4000`）
- 認証: 不要

### 5.2 新規エンドポイント

新規エンドポイントは不要（既存の `GET /products` を使用）。

## 6. データ設計

### 6.1 使用テーブル

#### books テーブル

| カラム名 | 型 | 用途 |
|---|---|---|
| id | INT (PK) | 一意識別子・ダミー評価算出に使用 |
| title | VARCHAR(200) | 表示 |
| author | VARCHAR(100) | 表示 |
| price | INT | 表示 |
| description | TEXT | 取得するが画面では未使用 |
| image_url | VARCHAR(500) | 表示 |
| created_at | TIMESTAMP | 未使用 |

並び順: `ORDER BY id`（登録順）。

### 6.2 CartContext のデータ構造

`addItem` に渡すオブジェクトは `Omit<CartItem, "quantity">` 型：

```typescript
addItem({ id: book.id, title: book.title, author: book.author, price: book.price })
```

`image_url` はカートに保持しない（`CartItem` 型に含まれないため）。

### 6.3 データフロー

```
[MySQL: books] → [Backend: GET /products] → [Frontend: page.tsx (useEffect)]
                                                      ↓
                                              [グリッド表示]
                                                      │
                                    「カートに追加」クリック
                                                      ↓
                                           [CartContext.addItem]
                                                      ↓
                                           [localStorage 永続化]
```

## 7. ビジネスロジック

- 書籍は `books` テーブルに存在する全件を表示する（在庫管理はスコープ外）。
- 並び順は ID 昇順（登録順）。
- ローディング中（API レスポンス待ち）は「読み込み中…」を表示する。
- 書籍一覧ページでも「カートに追加」ボタンを押せる（詳細ページへ遷移不要）。
- 星評価はダミーデータ（ID ベースの算出値）を表示する。実際の評価機能はスコープ外。

## 8. エラー処理

| エラー条件 | 現在の挙動 | 備考 |
|---|---|---|
| API通信失敗（ネットワークエラー） | `setLoading(false)` のみ。books が空配列のままグリッドが空表示される | エラーメッセージ・リトライボタンは未実装 |
| データ0件 | 空グリッドが表示される（空状態メッセージなし） | |
| 画像読み込み失敗 | ブラウザのデフォルト壊れ画像アイコン | `onError` ハンドラーは未実装 |

## 9. 非機能要件

- **パフォーマンス**:
  - 画像は外部CDN（placehold.co等）からの読み込み
- **セキュリティ**:
  - 認証不要の公開エンドポイントを使用
  - XSS対策: React のデフォルト挙動（テキストノードの自動エスケープ）に依拠
- **アクセシビリティ**:
  - 書影には `alt={book.title}` を設定

## 10. 制約・前提条件

- 検索・フィルター機能はスコープ外（検索バーはUI上存在するが機能しない）。
- ページング機能は未実装（書籍数が少ない前提）。
- 在庫管理はスコープ外のため、表示中の全書籍が常に注文可能とみなす。
- ログイン機能なしのため、誰でもアクセス可能。
- フロントエンドコンポーネントは `"use client"` ディレクティブが必須。
- スタイルはインラインスタイルのみ使用（CSSモジュール・Tailwind不可）。
