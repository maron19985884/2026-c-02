# 注文完了表示機能 外部設計書

## 1. 概要
- **機能ID**: F-05
- **機能名**: 注文完了表示機能
- **対象ユーザー**: 注文を確定した直後のユーザー
- **目的・背景**: 注文が正常に受け付けられたことをユーザーに明示し、注文番号を提示することで、後日問い合わせや注文管理ができるようにする。また、購入後の動線として商品一覧へ戻るリンクを提供し、回遊性を確保する。
- **作成日**: 2026-05-05
- **バージョン**: 1.1
- **関連要件**: U-16, U-17, U-18

## 2. 機能概要

注文フォーム（F-04）からの遷移で `/order/complete?orderNumber=ORD-xxxxxxxxxxxxx` というURLでアクセスされる。`useSearchParams()` でクエリパラメータ `orderNumber` を取得し、注文完了メッセージとともに画面上に表示する。画面下部に「書籍一覧に戻る」（黄色ボタン）と「注文履歴を確認する」（白ボタン、リンク先は `/`）を表示する。お届け予定情報（「Prime 対応 — 明日お届け予定」）と「確認メールをお送りします」メッセージも表示する。

Next.js 14 App Router の制約により、`useSearchParams()` を使用する `OrderCompleteContent` コンポーネントを `<Suspense>` でラップした2段構成をとる。

## 6. データ設計

### 6.1 使用テーブル

DBアクセスなし。

### 6.2 コンポーネント構成

Next.js 14 App Router の `useSearchParams` 制約から以下の2段構成をとる：

```
OrderCompletePage（サーバーコンポーネント的ラッパー）
  └─ <Suspense fallback="読み込み中…">
       └─ OrderCompleteContent（useSearchParams 使用）
```

### 6.3 データフロー

```
[注文フォーム]
   POST /orders 成功 → { order_number }
       ↓
   clearCart() → router.push('/order/complete?orderNumber=' + order_number)
       ↓
[注文完了画面]
   useSearchParams().get("orderNumber") → 画面表示
```

## 7. ビジネスロジック

### 表示処理

1. `useSearchParams().get("orderNumber")` で注文番号を取得
2. `orderNumber` が存在する場合: 完了メッセージ＋注文番号＋お届け予定を表示
3. `orderNumber` が `null` の場合: 注文番号表示セクションをスキップ（他のコンテンツは表示）

### 副次処理

- 表示時には `CartContext` の `items` は既に空（注文確定時に `clearCart()` 済み）
- ブラウザ「戻る」ボタンで F-04 に戻ると、カートが空のため「カートが空です」UI が表示される

## 8. エラー処理

| エラー条件 | 現在の挙動 |
|---|---|
| `orderNumber` クエリなし | 注文番号表示セクションをスキップして他コンテンツを表示 |
| `orderNumber` が想定形式外 | そのまま表示（バリデーションなし） |
| ブラウザの戻るで再表示 | 同じ画面が再表示される（注文番号は URL に残るため） |

## 9. 非機能要件

- **パフォーマンス**:
  - クライアント側のみで完結するため即時表示
- **セキュリティ**:
  - 注文番号はクエリパラメータに露出するが、DB照会なしのため情報漏洩なし
  - XSS対策: React の自動エスケープ（`dangerouslySetInnerHTML` 不使用）

## 10. 制約・前提条件

- 注文番号のフォーマットは `ORD-${Date.now()}`（13桁数値が末尾）。
- 本画面は注文情報の取得・参照API を持たない。
- 「注文履歴を確認する」ボタンはスコープ外機能のため `/` にリンクしている。
- お届け予定・確認メール送信は UI上の表示のみ（実際の処理なし）。
- Next.js 14 App Router の制約: `useSearchParams()` を使用するコンポーネントは `<Suspense>` 境界が必要。
- フロントエンドコンポーネントは `"use client"` ディレクティブが必須。
- スタイルはインラインスタイルのみ使用。
