# 商品詳細表示・カート追加機能 外部設計書

## 1. 概要
- **機能ID**: F-02
- **機能名**: 商品詳細表示・カート追加機能
- **対象ユーザー**: 書店サイトを訪れた購入希望者
- **目的・背景**: 個別書籍の詳細情報を提示し、購入意思の固まったユーザーがワンクリックでカートに追加できるようにする。商品理解の深化と購入意思決定を支援する画面。
- **作成日**: 2026-05-05
- **バージョン**: 1.0
- **関連要件**: U-04, U-05, U-06

## 2. 機能概要

商品一覧画面から特定書籍を選択するとこの画面に遷移する。書影・タイトル・著者・価格に加え、書籍の説明文（`description`）を表示する。「カートに追加」ボタンの押下で、カート（`CartContext` 経由・`localStorage` 永続化）に該当書籍を1冊追加する。追加後はユーザーが他の書籍を見続けられるよう、商品一覧へ戻る導線を提供する。

## 3. 画面・UI設計

### 3.1 画面一覧

| 画面名 | URL/パス | 説明 |
|---|---|---|
| 商品詳細画面 | `/products/[id]` | 個別書籍の詳細情報を表示。`[id]` は書籍ID（数値） |

### 3.2 画面レイアウト

#### 構成（2カラム or 縦並び）
- 左側（または上部）: 書影（大きめサイズ、例: 300x420px）
- 右側（または下部）: 書籍情報＋アクション領域

#### 表示項目

| 要素 | 内容 | スタイル要件 |
|---|---|---|
| 書影 | `image_url` | 大型表示 |
| タイトル | `title` | 見出しサイズ（h1 相当） |
| 著者 | `author` | サブテキスト |
| 価格 | `¥{price.toLocaleString()}` | 強調表示、ネイビー系 |
| 説明文 | `description` | 本文サイズ、改行を保持 |
| カートに追加ボタン | 「カートに追加」 | プライマリボタン（背景 `#1e3a8a`、白文字） |
| 一覧へ戻るリンク | 「← 商品一覧に戻る」 | テキストリンク |

#### スタイル仕様
- ページ背景: `#f3f4f6`
- カード背景: `white`、`box-shadow: 0 2px 8px rgba(0,0,0,0.08)`
- ボタン: 背景 `#1e3a8a`、ホバー時 `#1e40af`、白文字
- インラインスタイルのみ使用

### 3.3 画面遷移

```
[商品一覧 (/)]
     │ カードクリック
     ↓
[商品詳細 (/products/[id])]
     │
     ├─ 「カートに追加」 → カートに追加（同画面に留まる、または通知表示）
     ├─ 「商品一覧に戻る」 → [商品一覧 (/)]
     └─ ヘッダーのカート → [カート (/cart)]
```

## 4. 入出力定義

### 4.1 入力項目

#### URLパラメータ

| 項目名 | 型 | 必須 | バリデーション | 説明 |
|---|---|---|---|---|

### 4.2 出力項目

| 項目名 | 型 | 説明 |

| id | number | 書籍ID |
| title | string | 書籍タイトル |
| author | string | 著者名 |
| price | number | 価格（円） |
| description | string | 説明文（本機能で初めて表示） |

## 5. API設計

### 5.1 使用エンドポイント

| メソッド | パス | リクエスト | レスポンス | 説明 |
|---|---|---|---|---|
| GET | `/products/:id` | URLパス: `id`（数値） | `Book` | 単一書籍の詳細を取得 |

#### レスポンス例

```json
{
  "id": 1,
  "title": "プログラミング入門",
  "author": "山田 太郎",
  "price": 2800,
  "description": "プログラミングの基礎から丁寧に学べる入門書。変数・ループ・関数などの概念をわかりやすく解説します。",
  "image_url": "https://placehold.co/200x280/4A90E2/ffffff?text=Programming",
  "created_at": "2026-05-05T00:00:00.000Z"
}
```

#### 404ケース
- 該当 ID の書籍が存在しない場合: `404 { "error": "Not Found" }`

### 5.2 新規エンドポイント

新規エンドポイントは不要。

## 6. データ設計

### 6.1 使用テーブル

#### books テーブル
- F-01 と同様（全カラム表示。`description` を含む点が F-01 と異なる）。

#### CartContext（クライアント状態）

| プロパティ | 型 | 説明 |
|---|---|---|
| items | `CartItem[]` | カート内アイテム配列 |
| addItem | `(book: Book) => void` | 書籍を1冊追加（既存なら quantity+1） |

`CartItem = Book & { quantity: number }`

### 6.2 データフロー

```
[ユーザー] → URL `/products/[id]` でアクセス
    ↓
[Frontend] useParams で id 取得 → GET /products/:id
    ↓
[Backend] SELECT * FROM books WHERE id = ?
    ↓
[Frontend] state にセット → 詳細表示
    ↓
[ユーザー] 「カートに追加」クリック
    ↓
[CartContext.addItem(book)] → localStorage に永続化
```

## 7. ビジネスロジック

### 表示ロジック
1. URLパラメータ `id` を取得
2. `id` が数値変換可能か検証（不正な場合はエラー表示）
3. `GET /products/:id` を実行
4. 200応答: 詳細を表示／404応答: 「書籍が見つかりません」表示

### カート追加ロジック（`CartContext.addItem`）
1. 現在の `items` 配列を確認
2. 同一 `id` の書籍が既にあれば `quantity` を1増やす
3. なければ `{ ...book, quantity: 1 }` を新規追加
4. 更新後の配列を `localStorage` に保存

### 一覧へ戻るロジック
- Next.js の `Link` または `router.push("/")` で遷移
- カートの状態は `localStorage` 経由で保持されるため、戻っても消えない

## 8. エラー処理

| エラー条件 | エラーメッセージ | 対処方法 |
|---|---|---|
| `id` が数値でない | 「無効なURLです」 | 一覧へのリンクを表示 |
| 書籍が見つからない（404） | 「指定された書籍は存在しません」 | 一覧へのリンクを表示 |
| API通信失敗 | 「書籍情報の取得に失敗しました」 | リトライボタンまたは一覧リンクを表示 |
| サーバーエラー（500） | 同上 | 同上 |
| カート追加失敗（localStorage 容量超過等） | 「カートへの追加に失敗しました」 | エラー通知 |

## 9. 非機能要件

- **パフォーマンス**:
  - 初回表示: 2秒以内
  - カート追加処理: 100ms 以内（クライアント処理のみ）
- **セキュリティ**:
  - URLパラメータ `id` はバックエンドでパラメータバインディング使用済み（SQLインジェクション対策済み）
  - `description` は React の自動エスケープでXSS防止
- **アクセシビリティ**:
  - 書影に `alt={title}` を設定
  - 「カートに追加」ボタンは押下後にスクリーンリーダーへ通知（`aria-live` 等）
  - キーボード操作対応（Tab/Enter）

## 10. 制約・前提条件

- カートの永続化は `localStorage` のみ（サーバー側保存なし）。
- 同一書籍を連続で追加すると数量が加算される（独立アイテムにはならない）。
- 在庫管理はスコープ外のため、追加時に在庫数のチェックは行わない。
- ログイン不要のため、ユーザー識別なしで `localStorage` 単位のカートとなる（ブラウザを変えるとカート内容は引き継がれない）。
- フロントエンドコンポーネントは `"use client"` ディレクティブが必須（`useParams`, `useEffect`, `useContext` 使用のため）。
