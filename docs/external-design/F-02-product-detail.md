# 商品詳細表示・カート追加機能 外部設計書

## 1. 概要
- **機能ID**: F-02
- **機能名**: 商品詳細表示・カート追加機能
- **対象ユーザー**: 書店サイトを訪れた購入希望者
- **目的・背景**: 個別書籍の詳細情報を提示し、購入意思の固まったユーザーがワンクリックでカートに追加できるようにする。商品理解の深化と購入意思決定を支援する画面。
- **作成日**: 2026-05-05
- **バージョン**: 1.1
- **関連要件**: U-04, U-05, U-06

## 2. 機能概要

商品一覧画面から特定書籍を選択するとこの画面に遷移する。書影・タイトル・著者・価格・説明文・星評価を表示する。レイアウトは3カラム構成（左: 書影、中央: 商品情報、右: バイボックス）。バイボックスの「カートに追加」ボタン押下で CartContext 経由・localStorage 永続化にカート追加し、ボタンが「✓ カートに追加しました」に変化する（2.5秒後に元に戻る）。バイボックスには「カートを見る」ボタン（`/cart`）も表示する。カスタマーレビューセクションを画面下部に表示する（静的ダミーデータ）。

## 4. 入出力定義

### 4.1 入力項目

#### URLパラメータ

| 項目名 | 型 | 必須 | バリデーション | 説明 |
|---|---|---|---|---|
| id | string (数値文字列) | ○ | バックエンドに渡すのみ（フロントではバリデーションなし） | 書籍ID（`useParams()` 経由で取得） |

#### ユーザー操作

| 操作 | 内容 | 結果 |
|---|---|---|
| 「カートに追加」ボタン押下 | クリック | `addItem({ id, title, author, price })` 呼び出し、`added = true`（2.5秒後 false） |
| 「カートを見る」リンク押下 | クリック | `/cart` に遷移 |
| 「← 書籍一覧に戻る」リンク押下 | クリック | `/` に遷移 |

### 4.2 出力項目

| 項目名 | 型 | 説明 |
|---|---|---|
| id | number | 書籍ID |
| title | string | 書籍タイトル |
| author | string | 著者名 |
| price | number | 価格（円） |
| description | string | 説明文（本機能で初めて表示） |
| image_url | string | 書影画像URL |

#### カート追加時のフィードバック
- 「カートに追加」ボタンのラベルが「✓ カートに追加しました」に変化
- ボタン背景色が `#C3E6CB`（緑系）、テキスト色が `#155724` に変化
- 2.5秒後に元のスタイルに戻る
- ヘッダーのカートアイコンのバッジが即時更新

## 5. API設計

### 5.1 使用エンドポイント

| メソッド | パス | リクエスト | レスポンス | 説明 |
|---|---|---|---|---|
| GET | `/products/:id` | URLパス: `id` | `Book` | 単一書籍の詳細を取得 |

#### レスポンス例

```json
{
  "id": 1,
  "title": "プログラミング入門",
  "author": "山田 太郎",
  "price": 2800,
  "description": "プログラミングの基礎から丁寧に学べる入門書。",
  "image_url": "https://placehold.co/200x280/4A90E2/ffffff?text=Programming",
  "created_at": "2026-05-05T00:00:00.000Z"
}
```

#### 404ケース
- 該当 ID の書籍が存在しない場合: `404 { "error": "Not Found" }`

**注意**: 現在の実装では `res.ok` チェックを行わず直接 `res.json()` を呼んでいる。404 応答の場合 `{ error: "Not Found" }` が `book` state にセットされ、`!book` チェックで「書籍が見つかりません」が表示される（`book.id` 参照で実際はクラッシュする可能性がある）。

### 5.2 新規エンドポイント

新規エンドポイントは不要。

## 6. データ設計

### 6.1 使用テーブル

#### books テーブル
- F-01 と同様（全カラム表示。`description` を含む点が F-01 と異なる）。

#### CartContext（クライアント状態）

`addItem` に渡すオブジェクト型:
```typescript
Omit<CartItem, "quantity"> = { id: number; title: string; author: string; price: number }
```

`image_url` はカートに保持しない（`CartItem` 型に含まれない）。

### 6.2 データフロー

```
[ユーザー] → URL `/products/[id]` でアクセス
    ↓
[Frontend] useParams で id 取得 → GET /products/:id
    ↓
[Backend] SELECT * FROM books WHERE id = ?
    ↓
[Frontend] state にセット → 詳細表示
    ↓
[ユーザー] 「カートに追加」クリック
    ↓
[CartContext.addItem({ id, title, author, price })] → localStorage に永続化
    ↓
[ボタンラベル変化 → 2.5秒後に元に戻る]
```

## 7. ビジネスロジック

### 表示ロジック
1. URLパラメータ `id` を `useParams()` で取得
2. `GET /products/:id` を実行
3. 200応答: 詳細を表示／レスポンスが空の場合: 「書籍が見つかりません」表示

### カート追加ロジック（`CartContext.addItem`）
1. 現在の `items` 配列を確認
2. 同一 `id` の書籍が既にあれば `quantity` を1増やす
3. なければ `{ id, title, author, price, quantity: 1 }` を新規追加
4. 更新後の配列を `localStorage` に保存

### カスタマーレビュー
- 静的ダミーデータ2件を表示（DBには評価データなし）
- 評価: 固定で 4.2 / 5、1,234件

## 8. エラー処理

| エラー条件 | 現在の挙動 | 備考 |
|---|---|---|
| 書籍が見つからない（404） | `!book` のチェックで「書籍が見つかりません」表示 | `res.ok` チェックなし |
| API通信失敗（catch） | `setLoading(false)` のみ。`!book` で「書籍が見つかりません」表示 | |
| カート追加失敗（localStorage 容量超過等） | エラー通知なし | CartContext で try/catch なし |

## 9. 非機能要件

- **パフォーマンス**:
  - カート追加処理: クライアント処理のみ
- **セキュリティ**:
  - URLパラメータ `id` はバックエンドでパラメータバインディング使用済み（SQLインジェクション対策済み）
  - `description` は React の自動エスケープでXSS防止

## 10. 制約・前提条件

- カートの永続化は `localStorage` のみ（サーバー側保存なし）。
- 同一書籍を連続で追加すると数量が加算される。
- 在庫管理はスコープ外のため、「在庫あり」表示は常時。
- カスタマーレビュー・Prime情報はダミーデータ（スコープ外機能）。
- フロントエンドコンポーネントは `"use client"` ディレクティブが必須（`useParams`, `useEffect`, `useContext` 使用のため）。
- スタイルはインラインスタイルのみ使用。
