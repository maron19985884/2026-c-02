# カート管理機能 外部設計書

## 1. 概要
- **機能ID**: F-03
- **機能名**: カート管理機能
- **対象ユーザー**: 書店サイトを訪れた購入希望者（カートに1冊以上追加済みのユーザーを主とする）
- **目的・背景**: 購入予定の書籍を確認・編集できる場を提供する。数量変更・削除・合計金額のリアルタイム反映により、ユーザーが安心して注文手続きへ進めるようにする。
- **作成日**: 2026-05-05
- **バージョン**: 1.1
- **関連要件**: U-07, U-08, U-09, U-10, U-11

## 2. 機能概要

`/cart` にアクセスすると、`CartContext`（`localStorage` 永続化）に保持されている全アイテムを2カラムレイアウトで表示する（左: アイテム一覧、右: 注文サマリーパネル）。各アイテムについて、タイトル・著者・単価・数量・小計を表示し、セレクトボックス（1〜10）で数量変更可能。「削除」ボタンでカートから取り除ける。「レジに進む」ボタンで注文フォーム画面（`/order`）に遷移する。書影は `image_url` ではなく 📚 絵文字プレースホルダーを表示する。

## 4. 入出力定義

### 4.1 入力項目（ユーザー操作）

| 操作 | 入力項目 | バリデーション | 結果 |
|---|---|---|---|
| 数量セレクト変更 | 1〜10 の整数 | `quantity >= 1`（`updateQuantity` 内で `< 1` は無視） | `updateQuantity(id, value)` |
| 削除ボタン押下 | なし | なし | `removeItem(id)` |
| 「後で購入する」 | なし | なし | 現在は何もしない（UI上のみ） |
| 「レジに進む」 | なし | なし | `/order` へ遷移 |

### 4.2 出力項目（アイテム行ごと）

| 項目名 | 型 | 説明 |
|---|---|---|
| id | number | 書籍ID（key・`updateQuantity`/`removeItem` 引数） |
| title | string | 書名 |
| author | string | 著者名 |
| price | number | 単価 |
| quantity | number | 現在の数量（セレクトボックス初期値） |
| 小計（計算値） | number | `price × quantity` |

### 4.3 集計値

| 項目名 | 型 | 計算式 |
|---|---|---|
| 合計金額 (`total`) | number | `CartContext.total`（`Σ price × quantity`） |
| カート件数 (`cartCount`) | number | `items.reduce((sum, i) => sum + i.quantity, 0)` |

## 6. データ設計

### 6.1 使用データ

#### CartContext

| プロパティ | 型 | 説明 |
|---|---|---|
| items | `CartItem[]` | カート内アイテム配列 |
| updateQuantity | `(id: number, quantity: number) => void` | 数量変更（`< 1` の場合は無視） |
| removeItem | `(id: number) => void` | 削除 |
| total | number | 合計金額（計算済み） |

#### CartItem 型

```typescript
type CartItem = {
  id: number;
  title: string;
  author: string;
  price: number;
  quantity: number;
  // image_url は含まない
};
```

#### localStorage
- キー: `"cart"`
- 値: `CartItem[]` の JSON 文字列

### 6.2 データフロー

```
[localStorage: cart] ─→ [CartContext] → [/cart 画面表示]
                                               │
                           ┌─ 数量セレクト変更 → updateQuantity → localStorage 更新
                           ├─ 削除ボタン      → removeItem    → localStorage 更新
                           └─ 「レジに進む」  → /order へ遷移
```

## 7. ビジネスロジック

### 数量変更ロジック (`updateQuantity(id, quantity)`)
1. `quantity < 1` の場合は何もしない（セレクトボックスが1始まりのため実際には発生しない）
2. それ以外は対象 `id` の `quantity` を更新
3. `localStorage` に再保存

### 削除ロジック (`removeItem(id)`)
1. `items.filter(item => item.id !== id)` で配列を再構成
2. `localStorage` に再保存

### 合計金額 (`total`)
- `items.reduce((sum, i) => sum + i.price * i.quantity, 0)`
- `items` 変更時に再計算

### 「レジに進む」処理
1. カートが空でも「レジに進む」ボタンは非表示（空カート時は別UI）
2. `/order` へ遷移（`Link` コンポーネント）
3. カート内容は変更せずに保持

## 8. エラー処理

| エラー条件 | 現在の挙動 | 備考 |
|---|---|---|
| カートが空 | 「カートに商品がありません」＋「書籍一覧へ戻る」リンクを表示 | 「レジに進む」は非表示 |
| localStorage 読み書き失敗 | try/catch で握りつぶし、items は空配列のまま | エラーメッセージなし |

## 9. 非機能要件

- **パフォーマンス**:
  - 数量変更・削除・合計再計算: 同期的にクライアント上で実行
  - 初回表示: ローカル処理のみのため即時
- **セキュリティ**:
  - クライアント側のみで完結するため、特別なセキュリティ対策は不要
  - 注文確定時（F-04）のサーバー側では価格の再計算を行う

## 10. 制約・前提条件

- カートはブラウザの `localStorage` のみで永続化（サーバー側保存なし）。
- 数量上限はセレクトボックスで1〜10に固定（UIによる制限）。
- 書影（`image_url`）はカートに保持されないため表示不可。代わりに 📚 絵文字を表示。
- 送料は表示上「無料」（スコープ外のため計算なし）。
- 「後で購入する」ボタンは UI上存在するが機能しない。
- フロントエンドコンポーネントは `"use client"` ディレクティブが必須。
- スタイルはインラインスタイルのみ使用。
