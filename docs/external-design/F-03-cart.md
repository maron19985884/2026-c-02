# カート管理機能 外部設計書

## 1. 概要
- **機能ID**: F-03
- **機能名**: カート管理機能
- **対象ユーザー**: 書店サイトを訪れた購入希望者（カートに1冊以上追加済みのユーザーを主とする）
- **目的・背景**: 購入予定の書籍を確認・編集できる場を提供する。数量変更・削除・合計金額のリアルタイム反映により、ユーザーが安心して注文手続きへ進めるようにする。
- **作成日**: 2026-05-05
- **バージョン**: 1.0
- **関連要件**: U-07, U-08, U-09, U-10, U-11

## 2. 機能概要

`/cart` にアクセスすると、`CartContext`（`localStorage` 永続化）に保持されている全アイテムをテーブル形式で表示する。各アイテムについて、書名・単価・数量・小計を表示し、数量の増減ボタン（または直接入力）で変更可能。削除ボタンでカートから取り除ける。下部には全アイテムの合計金額が表示され、変更操作と同期してリアルタイムに更新される。「注文手続きへ進む」ボタンで注文フォーム画面（`/order`）に遷移する。

## 3. 画面・UI設計

### 3.1 画面一覧

| 画面名 | URL/パス | 説明 |
|---|---|---|
| カート画面 | `/cart` | カート内アイテムの確認・編集 |

### 3.2 画面レイアウト

#### 画面構成（カートにアイテムがある場合）

1. ページタイトル: 「ショッピングカート」
2. アイテム一覧テーブル
3. 合計金額表示
4. アクション領域（「注文手続きへ進む」ボタン、「買い物を続ける」リンク）

#### テーブル構成

| 列 | 内容 | 操作 |
|---|---|---|
| 書影（任意） | サムネイル画像 | なし |
| 書名 | `title` | なし |
| 単価 | `¥{price.toLocaleString()}` | なし |
| 数量 | 現在の `quantity` | `-` / `+` ボタン または数値入力 |
| 小計 | `price × quantity` | なし（自動計算） |
| 削除 | ゴミ箱アイコン or 「削除」ボタン | クリックでアイテム削除 |

#### 合計金額表示
- 「合計金額: ¥XX,XXX」形式
- 強調表示（大きめサイズ、ネイビー系色）
- アイテム変更時にリアルタイム更新

#### 空カート時の表示
- 「カートに商品がありません」メッセージ
- 「商品一覧に戻る」リンク
- 注文手続きボタンは非表示

#### スタイル仕様
- ページ背景: `#f3f4f6`
- テーブル背景: `white`、`box-shadow: 0 2px 8px rgba(0,0,0,0.08)`
- 削除ボタン: `#ef4444`（エラー・削除色）
- 「注文手続きへ進む」ボタン: 背景 `#1e3a8a`、白文字
- インラインスタイルのみ使用

### 3.3 画面遷移

```
[商品詳細] or [商品一覧のカートリンク]
              ↓
         [カート (/cart)]
              │
              ├─ 「注文手続きへ進む」 → [注文フォーム (/order)]
              └─ 「買い物を続ける」  → [商品一覧 (/)]
```

注: カート空の場合は「注文手続きへ進む」ボタンを非活性化または非表示。

## 4. 入出力定義

### 4.1 入力項目（ユーザー操作）

| 操作 | 入力項目 | バリデーション | 結果 |
|---|---|---|---|
| 数量変更（+ボタン） | アイテムID | なし | `quantity + 1` |
| 数量変更（-ボタン） | アイテムID | `quantity > 1` 時のみ動作（または1未満で削除） | `quantity - 1` |
| 数量直接入力（任意実装） | 数値 | 1以上の整数 | 入力値で `quantity` 更新 |
| 削除ボタン押下 | アイテムID | なし | アイテムをカートから削除 |
| 「注文手続きへ進む」 | なし | カートに1件以上 | `/order` へ遷移 |

### 4.2 出力項目（テーブル行ごと）

| 項目名 | 型 | 説明 |
|---|---|---|
| id | number | 書籍ID（key として使用） |
| title | string | 書名 |
| price | number | 単価 |
| quantity | number | 数量 |
| 小計（計算値） | number | `price × quantity` |
| image_url | string | 書影URL（表示する場合） |

### 4.3 集計値

| 項目名 | 型 | 計算式 |
|---|---|---|
| 合計金額 (`total`) | number | `Σ (price × quantity)` 全アイテム |
| アイテム種別数 | number | `items.length` |

## 5. API設計

### 5.1 使用エンドポイント

本画面はクライアントサイドの `CartContext` および `localStorage` のみで完結し、API呼び出しは行わない。

### 5.2 新規エンドポイント

不要。

## 6. データ設計

### 6.1 使用データ

#### CartContext

| プロパティ | 型 | 説明 |
|---|---|---|
| items | `CartItem[]` | カート内アイテム配列 |
| addItem | `(book: Book) => void` | 追加（本画面では未使用） |
| updateQuantity | `(id: number, quantity: number) => void` | 数量変更 |
| removeItem | `(id: number) => void` | 削除 |
| clearCart | `() => void` | 全削除（注文確定時に使用） |
| total | number | 合計金額（計算済み） |

#### localStorage

- キー: `cart`（実装依拠）
- 値: `CartItem[]` の JSON 文字列

### 6.2 データフロー

```
[localStorage: cart] ─┐
                      ├─→ [CartContext] → [/cart 画面表示]
                      │                          │
                      │            ┌─ 数量変更 / 削除
                      │            ↓
                      └─← [CartContext 更新]
                                   │
                                   └─ localStorage に再永続化
```

すべての変更は同期的にクライアント上で行われ、即座に画面に反映される。

## 7. ビジネスロジック

### 数量変更ロジック (`updateQuantity(id, quantity)`)
1. `items` 配列内の該当 `id` のアイテムを検索
2. `quantity` が 0 以下の場合、`removeItem(id)` を呼び出す（または最小値1で固定）
3. それ以外は `quantity` を更新
4. `localStorage` に再保存

### 削除ロジック (`removeItem(id)`)
1. `items.filter(item => item.id !== id)` で配列を再構成
2. `localStorage` に再保存

### 合計金額計算 (`total`)
- `items.reduce((sum, item) => sum + item.price * item.quantity, 0)`
- `items` 変更時に再計算（`useMemo` 推奨）

### 注文手続きへ進む処理
1. カートが空でないことを確認
2. `/order` へ遷移（`Link` または `router.push`）
3. カート内容は変更せずに保持（注文確定後に `clearCart` を呼ぶのは F-04 の責務）

## 8. エラー処理

| エラー条件 | エラーメッセージ | 対処方法 |
|---|---|---|
| カートが空 | 「カートに商品がありません」 | 空状態UIを表示、注文ボタンを無効化／非表示、商品一覧リンクを提示 |
| 数量を0以下に変更 | （メッセージなし） | アイテムを自動削除する、または最小値1で固定（実装方針による） |
| localStorage 読み書き失敗 | 「カート情報の保存に失敗しました」 | エラー通知、操作続行は可能（メモリ上の state は機能） |
| `localStorage` 利用不可（ブラウザ設定） | 「カート機能はブラウザ設定によりご利用いただけません」 | 警告を表示、メモリ上のみで動作 |

## 9. 非機能要件

- **パフォーマンス**:
  - 数量変更・削除・合計再計算: 100ms 以内
  - 初回表示: ローカル処理のみのため即時（< 500ms）
- **セキュリティ**:
  - クライアント側のみで完結するため、特別なセキュリティ対策は不要
  - ただし注文確定時（F-04）にはサーバー側でも価格を検証することを推奨（本機能スコープ外）
- **アクセシビリティ**:
  - 数量増減ボタンに `aria-label` を設定（例: `aria-label="数量を1増やす"`）
  - 削除ボタンに `aria-label="〇〇を削除"` を設定
  - テーブルには適切な `<th scope="col">` を使用
  - 合計金額の更新は `aria-live="polite"` で通知

## 10. 制約・前提条件

- カートはブラウザの `localStorage` のみで永続化（サーバー側保存なし）。
- ブラウザを変える、シークレットモード、Cookie/Storage クリア等でカートは失われる。
- 在庫管理はスコープ外のため、数量上限のチェックは行わない（任意の正の整数を許容）。
- 送料・税・割引等の計算はスコープ外（要件 U-10 の補足条件「送料などの表示ルールはスコープ外」）。
- フロントエンドコンポーネントは `"use client"` ディレクティブが必須。
- スタイルはインラインスタイルのみ使用。
