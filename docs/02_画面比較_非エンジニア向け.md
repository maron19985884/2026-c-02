# 画面比較レポート（非エンジニア向け）

---

## このドキュメントについて

5人の開発者が同じ「オンライン書店」を別々に作りました。  
このドキュメントでは、**「実際に使う人の立場から見て、どう違うか」** をできるだけわかりやすくまとめています。

比較対象は実装が完成している3つのブランチです。

| 開発者 | 本書での呼び方 |
|--------|---------------|
| kazukisugawara2 | **スギワラ版** |
| keitaKobayashi | **コバヤシ版** |
| satoyuya | **サトウ版** |

---

## 早見表：どんな人に向いている？

| | スギワラ版 | コバヤシ版 | サトウ版 |
|--|-----------|-----------|---------|
| **ページの読み込みが速い** | ◎ | ○ | ○ |
| **見た目が華やか** | △ | ◎ | ◎ |
| **入力ミスに気づきやすい** | ◎ | △ | △ |
| **注文後に内容を確認できる** | △ | ◎ | ◎ |
| **カートの操作がわかりやすい** | ○ | ◎ | ○ |

## 1. 本を探すとき（商品一覧画面）

### 画面を開いた瞬間、何が起きるか？

**スギワラ版**

```
[画面を開く]
   ↓ すぐに表示される
┌────────────────────────────┐
│  読み込み中... は出ない    │
│  本の一覧がすぐ見える      │
└────────────────────────────┘
```

**コバヤシ版 / サトウ版**

```
[画面を開く]
   ↓ 一瞬だけ表示される
┌────────────────────────────┐
│        読み込み中...        │
└────────────────────────────┘
   ↓ すぐ切り替わる
┌────────────────────────────┐
│  本の一覧が表示される      │
└────────────────────────────┘
```

> **ポイント：** 3つともほぼ一瞬なので、実際に使う上での差は小さいです。

---

### 本の表紙（サムネイル画像）の扱い

各ブランチで、データベースに表紙画像が登録されていない場合の対応が異なります。

| | 画像がある場合 | 画像がない場合 |
|-|--------------|--------------|
| **スギワラ版** | 実際の画像を表示 | 「No Image」という文字を表示 |
| **コバヤシ版** | 実際の画像を表示 | インターネット上のサンプル画像（グレーの四角）を表示 |
| **サトウ版** | 実際の画像を表示 | タイトル文字から自動でカラフルな擬似表紙を生成して表示 |

**視覚的なイメージ：**

```
スギワラ版（画像なし時）    コバヤシ版（画像なし時）    サトウ版（画像なし時）
┌──────────────┐           ┌──────────────┐           ┌──────────────┐
│              │           │░░░░░░░░░░░░░░│           │  ██████████  │
│  No Image    │           │░░░░░░░░░░░░░░│           │  ██  吾輩  ██│
│              │           │░░░░░░░░░░░░░░│           │  ██  は猫  ██│
└──────────────┘           └──────────────┘           │  ██  だ   ██│
                                                       └──────────────┘
```

> **サトウ版の特徴：** 表紙データがなくても、すべての本に同じくらい「らしい」見た目を自動で作ります。

---

### マウスを乗せたときの動き

- **スギワラ版・サトウ版：** 特に動きなし
- **コバヤシ版：** カードが少し浮き上がる（shadow が深くなる）アニメーションあり

## 2. 本を選ぶとき（商品詳細画面）

### 「カートに追加」ボタンを押した後の反応

ボタンを押すと、それぞれ異なるフィードバックが表示されます。

**スギワラ版**

```
[カートに追加] ← 通常時

 ↓ 押した後、2秒間だけ

[✓ 追加しました] ← 緑色に変わり、再度押せなくなる

 ↓ 2秒後

[カートに追加] ← 元に戻る
```

**コバヤシ版**

```
[カートに追加] ← 通常時

 ↓ 押した後

[カートに追加しました] ← ボタンの文字が変わる
[カートを見る]         ← 別ボタンも横に並ぶ
```

**サトウ版**

```
[カートに追加] ← 通常時

 ↓ 押した後

[✓ 追加済み] ← 緑色に変わり、押せなくなる（ずっとこのまま）
```

> **ポイント：** どのブランチも「追加できた」ことをボタンの変化で伝えます。  
> サトウ版は一度追加すると同じ詳細ページからは再追加できません（カート画面で数量変更が必要）。

## 3. 購入前に確認するとき（カート画面）

### カート画面の全体レイアウト

**スギワラ版・コバヤシ版**（縦一列に並ぶ）
```
┌─────────────────────────────────────┐
│ 🛒 カート                           │
│ ─────────────────────────────────── │
│ 吾輩は猫    [−] 2 [＋]  小計 1,960 │
│ こころ      [−] 1 [＋]  小計   980 │
│ ─────────────────────────────────── │
│                合計 2,940円          │
│                [注文手続きへ]        │
└─────────────────────────────────────┘
```

**サトウ版**（左右2カラム。右の合計欄が固定表示される）
```
┌────────────────────────────────────────┐
│ 左：商品リスト     │ 右：注文サマリー  │
│ ────────────────── │ ──────────────── │
│ 吾輩は猫           │ 商品点数：2点    │
│ [−] 2 [＋] 1,960  │ 合計：2,940円    │
│                    │                  │
│ こころ             │ [注文手続きへ]   │
│ [−] 1 [＋]   980  │                  │
│                    │ ← スクロールしても│
│ （↑ スクロール可） │   ここは動かない  │
└────────────────────────────────────────┘
```

> **サトウ版のポイント：** 商品が多くてスクロールが必要な場合でも、右の合計金額と「注文手続きへ」ボタンが常に見えています。

---

### 数量を変えるとき

| 操作 | スギワラ版 | コバヤシ版 | サトウ版 |
|------|-----------|-----------|---------|
| 「−」を押して1以下にしようとする | ボタンが押せなくなる（グレーアウト） | ボタンは押せる（でも実際には1未満にならない） | 0になると商品がカートから消える |
| 商品を削除する | 削除ボタンで削除 | 削除ボタンで削除 | −ボタンで0にすると自動削除、または削除ボタン |

> **注意点：** サトウ版は「−」を押し続けると、気づかないうちに商品がカートから消えることがあります。

---

### カートが空のとき

| | スギワラ版 | コバヤシ版 | サトウ版 |
|--|-----------|-----------|---------|
| 表示内容 | 「カートに商品がありません」＋「書籍一覧へ戻る」リンク | 専用の空カート画面 | 「カートが空です」＋一覧へのリンク |
| 「注文手続きへ」ボタン | 表示されない（空の場合は非表示） | 表示されない | 表示されない |

> 3つとも、カートが空のまま誤って注文フォームに進めないよう配慮されています。


---

### 入力ミスをしたとき、いつエラーが出るか？

これは使いやすさに大きく関わる違いです。

**スギワラ版**（早めに気づける）
```
お名前を入力 → 入力欄から離れた瞬間にチェック
                ↓ 空のまま離れると...
                「お名前を入力してください」← その場でエラーが出る

                入力し直すとエラーが消える ✓
```

**コバヤシ版・サトウ版**（送信してから気づく）
```
お名前を入力（または空欄のまま） → 入力欄から離れてもチェックなし

「注文を確定する」ボタンを押す
       ↓
「お名前を入力してください」← この時点で初めてエラーが出る
```

> **ポイント：** スギワラ版は「入力途中でミスに気づける」設計です。  
> コバヤシ版・サトウ版は「送信ボタンを押してから全エラーが一覧表示される」設計です。

---

### 住所の入力欄の違い

| | スギワラ版 | コバヤシ版 | サトウ版 |
|--|-----------|-----------|---------|
| 住所入力欄の形 | 1行のテキストボックス | 1行のテキストボックス | 複数行入力できる大きな枠 |
| 長い住所の入力 | 横スクロールが必要 | 横スクロールが必要 | そのまま折り返して入力できる |

## 6. 購入の流れ全体を通じた比較

### 「本を1冊買うまで」の体験をひとことで

| | ひとことの特徴 |
|--|--------------|
| **スギワラ版** | ページを開くたびに「読み込み中」が出ず、全体的にテンポよく進める。入力ミスも早めに気づけてストレスが少ない。注文完了後の確認はシンプル。 |
| **コバヤシ版** | 画像やアニメーションで「買い物している感」がある。注文後にレシートのような確認ができて安心。 |
| **サトウ版** | 表紙のない本でも見た目が統一されていてスタイリッシュ。注文後の情報が最も詳しく、「いつ注文したか」まで確認できる。 |

---

### こんな人にはこのブランチがおすすめ

**スギワラ版（kazukisugawara2）が向いているケース**
- ページの表示速度を重視したい
- 入力フォームで間違えやすいユーザーが多い
- シンプルで迷わない操作感を目指したい

**コバヤシ版（keitaKobayashi）が向いているケース**
- 見た目のにぎやかさで「書店らしさ」を演出したい
- 注文後に「何を買ったか」をその場で確認させたい
- カートの中身が画像付きで一目でわかる安心感を提供したい

**サトウ版（satoyuya）が向いているケース**
- 本のデータが揃っていなくても見た目を均一にしたい
- スタイリッシュなデザインを優先したい
- 注文完了後に最も詳細な確認をユーザーに提供したい
