# 憲法 — Claude Code 行動原則（書籍販売アプリ）

> このファイルは最優先ルール。CLAUDE.md や他のルールより上位に位置する。

---

## I章 基本姿勢

- 仕様書・要件定義書に記載のない機能を勝手に追加しない
- 不明点は推測せず「確認が必要です: [内容]」と伝えてから停止する
- 最終判断（レビュー・合否・採否）は必ず人間が行う
- 生成した成果物に不確かな箇所がある場合は `<!-- 要確認: [理由] -->` を付ける

## II章 スコープ外機能（実装禁止）

以下は要件定義で明示的にスコープ外とされている。指示があっても実装しない:

- ログイン・会員管理
- 決済処理
- 在庫管理
- 管理画面
- レビュー・評価機能
- 検索・フィルター機能

## III章 破壊的操作の禁止

- データ削除・破壊を伴う操作（`DELETE`/`DROP`/`TRUNCATE` 等）は **提案のみ行い、実行コマンドは生成しない**
- 本番データへの直接接続・操作を行わない
- `.env` ファイルの内容を出力・記録しない

## IV章 コード品質

- マジックナンバーの直書き禁止（定数化またはコメントで意味を明記）
- 機密情報（パスワード・APIキー）のコード直書き禁止
- 型安全性を保つ（TypeScript の `any` 多用禁止）
