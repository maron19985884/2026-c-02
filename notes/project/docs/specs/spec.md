## 1. 機能概要

個人運営オンライン書店の購買フロー全体。5つの画面で構成される。

| 画面ID | 画面名 | 対応要件 |
|---|---|---|
| SCR-01 | 商品一覧 | REQ-001〜003 |
| SCR-02 | 商品詳細 | REQ-004〜006 |
| SCR-03 | カート | REQ-007〜011 |
| SCR-04 | 注文フォーム | REQ-012〜015 |
| SCR-05 | 注文完了 | REQ-016〜018 |

---

## 2. データモデル

### 書籍（Book）

| フィールド | 型 | 説明 |
|---|---|---|
| id | number | 書籍ID（主キー） |
| title | string | タイトル |
| author | string | 著者名 |
| price | number | 価格（円、税込） |
| imageUrl | string | 書影URL |
| description | string | 説明文 |

### カートアイテム（CartItem）

| フィールド | 型 | 説明 |
|---|---|---|
| bookId | number | 書籍ID（Book への参照） |
| quantity | number | 数量（1以上の整数） |

### 注文（Order）

| フィールド | 型 | 説明 |
|---|---|---|
| orderId | string | 注文番号（UUID など） |
| customerName | string | 氏名 |
| address | string | 住所 |
| email | string | メールアドレス |
| items | CartItem[] | 注文書籍一覧 |
| totalAmount | number | 合計金額 |

## 3. 画面別仕様

### SCR-01 商品一覧

**入力:** なし（初期表示時に全書籍を取得）

**出力:**

| 項目 | 表示内容 |
|---|---|
| 書籍リスト | 書影・タイトル・著者・価格をグリッド形式で表示 |

**処理フロー（正常系）:**
1. ページ表示時に `GET /api/books` を呼び出す
2. レスポンスの書籍一覧をグリッド表示する
3. 書籍カードをクリックすると SCR-02（商品詳細）へ遷移する

**処理フロー（異常系）:**

| ケース | 対応 |
|---|---|
| API エラー | エラーメッセージを表示する |
| 書籍が0件 | 「現在販売中の書籍はありません」と表示する |

---

### SCR-02 商品詳細

**入力:** URL パラメータ `bookId`

**出力:**

| 項目 | 表示内容 |
|---|---|
| 書籍情報 | 書影・タイトル・著者・価格・説明文 |
| ボタン | 「カートに追加」「一覧へ戻る」 |

**処理フロー（正常系）:**
1. `bookId` を元に `GET /api/books/:id` を呼び出す
2. 書籍情報を表示する
3. 「カートに追加」押下 → カートに追加してトースト等で通知、画面は移動しない
4. 「一覧へ戻る」押下 → SCR-01 へ遷移

**処理フロー（異常系）:**

| ケース | 対応 |
|---|---|
| 書籍が見つからない | 「該当の書籍が見つかりません」を表示し一覧へ戻るリンクを出す |

---

### SCR-03 カート

**入力:** カート状態（フロントエンドの state）

**出力:**

| 項目 | 表示内容 |
|---|---|
| カート一覧 | 書名・単価・数量（増減ボタン）・小計 |
| 合計金額 | 全アイテムの小計の総和 |
| ボタン | 「注文手続きへ」「削除（各行）」 |

**処理フロー（正常系）:**
1. カート状態から書籍情報を表示する
2. 数量増減ボタン押下 → 数量・小計・合計をリアルタイム更新
3. 削除ボタン押下 → 該当書籍をカートから除去し合計を更新
4. 「注文手続きへ」押下 → SCR-04 へ遷移

**業務ルール:**
- 数量は 1 以上。1 の状態で「－」は押せない（または押すと削除）
- 小計 = 単価 × 数量
- 合計 = 全アイテム小計の総和（送料は含まない）

**処理フロー（異常系）:**

| ケース | 対応 |
|---|---|
| カートが空 | 「カートに商品がありません」を表示し一覧へ戻るリンクを出す |
| 空カートで注文手続きへ | ボタンを非活性またはエラー表示 |

---

### SCR-04 注文フォーム

**入力:**

| 項目名 | 型 | 必須 | バリデーション |
|---|---|---|---|
| 氏名 | string | ○ | 1文字以上 |
| 住所 | string | ○ | 1文字以上 |
| メールアドレス | string | ○ | RFC準拠のメール形式 |

**出力:**

| 項目 | 表示内容 |
|---|---|
| 注文内容 | カート内書籍と合計金額（読み取り専用） |
| フォーム | 氏名・住所・メール入力欄 |
| エラー | 各項目のバリデーションエラーメッセージ |
| ボタン | 「注文する」 |

**処理フロー（正常系）:**
1. カートの注文内容（書籍一覧・合計）を表示する
2. ユーザーが氏名・住所・メールを入力する
3. 「注文する」押下 → バリデーション実行
4. バリデーション通過 → `POST /api/orders` を呼び出す
5. 成功 → SCR-05（注文完了）へ遷移し注文番号を渡す

**処理フロー（異常系）:**

| ケース | 対応 |
|---|---|
| 必須項目が未入力 | 該当項目の下にエラーメッセージを表示、送信しない |
| メール形式不正 | 「正しいメールアドレスを入力してください」を表示 |
| API エラー | エラーメッセージを表示し、フォームは入力状態を維持 |

---

### SCR-05 注文完了

**入力:** 注文番号（SCR-04 からの遷移データ）

**出力:**

| 項目 | 表示内容 |
|---|---|
| 完了メッセージ | 「ご注文ありがとうございました」等 |
| 注文番号 | 発行された注文番号 |
| リンク | 「商品一覧へ戻る」 |

**処理フロー（正常系）:**
1. 注文番号を表示する
2. 「商品一覧へ戻る」押下 → SCR-01 へ遷移し、カートをクリアする

## 4. API 仕様

| メソッド | パス | 概要 | リクエスト | レスポンス |
|---|---|---|---|---|
| GET | `/api/books` | 書籍一覧取得 | なし | `Book[]` |
| GET | `/api/books/:id` | 書籍詳細取得 | パスパラメータ `id` | `Book` |
| POST | `/api/orders` | 注文作成 | `Order`（`orderId` 除く） | `{ orderId: string }` |
