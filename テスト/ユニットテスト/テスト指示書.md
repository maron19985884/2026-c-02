# ユニットテスト 指示書

## 1. テスト概要

| 項目 | 内容 |
|------|------|
| テスト種別 | ユニットテスト |
| 対象システム | オンライン書店アプリケーション |
| テスト実施日 | 2026-05-13 |
| テストフレームワーク | Jest 29 + ts-jest 29 |
| 総テストケース数 | 51件 |

## 2. テスト目的

個々の関数・ロジックが単体で正しく動作することを確認する。
外部依存（データベース、外部APIなど）を完全に排除し、純粋なビジネスロジックのみを検証する。

## 3. テスト対象と構成

```
ユニットテスト/
├── backend/
│   ├── generateOrderNumber.test.ts  # 注文番号生成ロジック（4件）
│   └── validation.test.ts           # 注文バリデーションロジック（20件）
└── frontend/
    ├── cartReducer.test.ts          # カートReducer（17件）
    └── checkoutValidation.test.ts   # チェックアウトバリデーション（10件）
```

## 6. テスト環境

### 前提条件
- Node.js 20.x 以上
- npm 10.x 以上
- インターネット接続（npm install 用）

### データベース・サーバー不要
ユニットテストは外部依存なしで実行可能。

---

## 7. セットアップ手順

```bash
cd テスト/ユニットテスト
npm install
```

---

## 8. テスト実行方法

### 全テスト実行
```bash
npm test
```

### Backend テストのみ実行
```bash
npm run test:backend
```

### Frontend テストのみ実行
```bash
npm run test:frontend
```

---

## 9. 合格基準

- 全51テストケースが PASS すること
- テスト実行中にコンパイルエラーが発生しないこと
- 各バリデーション関数のエラーメッセージ文言が完全一致すること
