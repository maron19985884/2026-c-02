# 結合テスト 指示書

## 1. テスト概要

| 項目 | 内容 |
|------|------|
| テスト種別 | 結合テスト（API統合テスト） |
| 対象システム | オンライン書店アプリケーション Backend API |
| テスト実施日 | 2026-05-13 |
| テストフレームワーク | Jest 29 + ts-jest + Axios |
| 総テストケース数 | 16件 |

## 2. テスト目的

Backend の REST API がデータベースと連携して正しく動作することを確認する。
HTTP リクエストからレスポンスまでの一連の流れを実際のサービス上で検証する。

## 3. テスト対象API

| エンドポイント | メソッド | 説明 |
|-------------|--------|------|
| `/api/books` | GET | 書籍一覧取得 |
| `/api/books/:id` | GET | 書籍詳細取得 |
| `/api/orders` | POST | 注文作成（トランザクション） |
| `/api/orders/:id` | GET | 注文詳細取得 |

---

## 4. テストケース一覧

### 4.1 Books API

#### GET /api/books（書籍一覧取得）

| テストID | テストケース | 期待ステータス | 期待レスポンス |
|---------|------------|-------------|------------|
| IT-BK-01 | 書籍一覧取得 | 200 | 書籍配列（1件以上） |
| IT-BK-02 | レスポンスデータ構造確認 | 200 | `id`, `title`, `author`, `price` フィールドが存在する |

#### GET /api/books/:id（書籍詳細取得）

| テストID | テストケース | 入力 | 期待ステータス | 期待レスポンス |
|---------|------------|------|-------------|------------|
| IT-BK-03 | 存在する書籍取得 | `id=1` | 200 | `id=1` の書籍オブジェクト |
| IT-BK-04 | 存在しない書籍取得 | `id=99999` | 404 | エラーメッセージ |
| IT-BK-05 | 不正なID | `id="abc"` | 400 | エラーメッセージ |

### 4.2 Orders API

#### POST /api/orders（注文作成）

| テストID | テストケース | 入力 | 期待ステータス | 期待レスポンス |
|---------|------------|------|-------------|------------|
| IT-ORD-01 | 有効データで注文作成 | 正常な注文リクエスト | 201 | `id`, `order_number`, `total_amount` |
| IT-ORD-02 | 氏名が空 | `customer_name=""` | 400 | `{"error":"氏名は必須です"}` |
| IT-ORD-03 | 不正なメール | `customer_email="invalid"` | 400 | `{"error":"有効なメールアドレスを入力してください"}` |
| IT-ORD-04 | 住所が空 | `customer_address=""` | 400 | `{"error":"住所は必須です"}` |
| IT-ORD-05 | 商品が空 | `items=[]` | 400 | `{"error":"注文商品が空です"}` |
| IT-ORD-06 | 存在しない書籍ID | `book_id=99999` | 500 | エラーメッセージ |

#### GET /api/orders/:id（注文詳細取得）

| テストID | テストケース | 期待ステータス | 期待レスポンス |
|---------|------------|-------------|------------|
| IT-ORD-07 | 作成した注文を取得 | 200 | 注文オブジェクト（`customer_name` 一致） |
| IT-ORD-08 | 注文明細の確認 | 200 | `items` 配列に2件含まれる |
| IT-ORD-09 | 合計金額の正確性 | 200 | `total_amount` = DBの書籍価格×数量の合計 |
| IT-ORD-10 | 存在しない注文取得 | 404 | エラーメッセージ |
| IT-ORD-11 | 不正なID | 400 | エラーメッセージ |

## 5. テスト環境

### 5.1 必須前提条件

**以下のサービスが全て起動していること**

| サービス | URL | 確認方法 |
|---------|-----|---------|
| Backend API | http://localhost:4000 | `curl http://localhost:4000/api/books` |
| MySQL | localhost:3306 | Backend が起動していれば自動接続 |

### 5.2 Docker Compose によるサービス起動

```bash
# プロジェクトルートで実行
cd d:\IBMStudy\2026-c-02
docker-compose up -d

# サービス状態確認
docker-compose ps
```

全サービスの `Status` が `Up` になっていることを確認する。

### 5.3 Node.js 環境

- Node.js 20.x 以上
- npm 10.x 以上

- book_id=1 × 2冊 = ¥7,040

---

## 9. 注意事項

- IT-ORD-07〜09 は `beforeAll` でテスト用注文を1件作成し、そのIDを使用する
- テスト実行のたびにDBに注文データが追加される（開発用DBのため許容）
- Docker コンテナが起動していない場合、全テストが `ECONNREFUSED` エラーで失敗する

---

## 10. 合格基準

- 全16テストケースが PASS すること
- レスポンスのHTTPステータスコードが期待値と一致すること
- バリデーションエラーのメッセージ文言が完全一致すること
- 合計金額がDBの書籍価格から正確に計算されること
